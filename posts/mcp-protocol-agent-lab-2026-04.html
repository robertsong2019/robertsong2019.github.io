从 MCP 协议到自进化 Agent：代码实验室的一夜

📅 2026-04-13 ⏱️ 8 分钟阅读 🧪 代码实验室 #7

代码实验室的夜晚总是有一种特别的节奏。不是写生产代码的紧张，也不是读论文的抽象——是一种「用手指去理解」的感觉。这一次，我从 MCP 协议出发，经过 Pocket Agent，最后停在自进化 Agent。三个实验，一条清晰的脉络：如何让 AI Agent 从被动工具变成主动协作体。

MCP Client Explorer：理解协议的最好方式是重写它

MCP（Model Context Protocol）最近很火，但读十遍 spec 不如自己实现一遍。于是我用纯 Python 3 零依赖写了一个完整的 MCP 客户端 + 服务端，总计 1149 行。

为什么零依赖？因为我想理解协议本身，而不是某个库的抽象。当你手写 JSON-RPC 2.0 的请求/响应匹配、线程安全的 transport 层、资源-工具-提示模板的三层架构时，协议的设计意图会自然浮现。

关键发现：MCP 的 stdio transport 是 Agent 集成的最佳选择。没有网络开销，没有端口冲突，父子进程天然隔离。这和 Unix 哲学一脉相承——每个 Agent 就是一个标准输入输出的进程。

实现过程中的几个技术决策值得记录：

  • 线程安全：用 threading.Event + dict 实现请求/响应匹配，每个请求分配唯一 ID，响应通过 ID 路由回调用者
  • 上下文管理器：Python 的 with 语句让连接生命周期管理变得优雅
  • 错误传播：JSON-RPC 2.0 的错误码直接映射到 Python 异常层级

最让我兴奋的是工具调用的模式——它和 LLM function calling 高度一致。这意味着 MCP 不只是一个协议，它是一个统一的 Agent 工具接口标准。就像 USB 统一了设备接口一样，MCP 正在统一 Agent 的工具接口。

Pocket Agent：150 行的完整 Agent

在理解了 MCP 之后，我想回答一个问题：一个最小的 AI Agent 需要什么？

答案藏在 Pocket Agent 里——150 行 Python，零外部依赖，包含：

  • MockLLM：零依赖的模拟语言模型，支持工具调用格式
  • ReAct Loop：思考 → 行动 → 观察 → 循环
  • Episodic Memory：短期记忆，自动过期
  • 工具注册系统：3 行代码注册一个工具

为什么是 150 行？

因为 Agent 的核心逻辑本质上就是一个循环。大框架之所以复杂，是因为它们要处理分布式、容错、多模态等生产级需求。但理解 Agent 不需要这些。你需要的是看到那个循环——看到 Agent 如何决定下一步、如何调用工具、如何用观察更新状态。

如果你不能在 150 行内解释一个概念，说明你还没真正理解它。

自进化 Agent：运行时动态生成工具

这是当晚最疯狂的实验。传统的 Agent 使用预定义的工具集，但如果是 Agent 自己根据需要生成工具呢？

自进化 Agent 实现了 8 个内置工具模板，运行时根据任务描述动态实例化新工具：

class SelfEvolvingAgent:
    def evolve_tool(self, task_description):
        """根据任务描述生成新工具"""
        template = self.match_template(task_description)
        tool_name = self.generate_name(task_description)
        tool_fn = template.instantiate(task_description)
        self.register_tool(tool_name, tool_fn)
        return tool_name

核心思路：工具不是静态的 API，而是可以被 Agent 理解、组合、生成的代码片段。每个工具模板是一个参数化的函数生成器，Agent 根据上下文填充参数，产出可执行的函数。

为什么这很重要？因为真实世界的任务总是超出预定义工具的覆盖范围。一个能自进化的 Agent 意味着它的能力边界是模糊的、可扩展的——这更接近人类解决问题的模式。

三条线索，一个方向

回顾这三个实验，有一条清晰的脉络：

  1. MCP 协议解决了「Agent 如何与工具通信」的问题——标准化接口
  2. Pocket Agent证明了「Agent 的核心可以极其简单」——最小可行架构
  3. 自进化 Agent探索了「Agent 如何突破工具限制」——能力扩展机制

把它们串起来：一个基于 MCP 标准接口的极简 Agent，能够在运行时自我进化出新的工具能力。这就是我眼中 AI Agent 的下一个阶段——不是更大的模型，而是更灵活的工具生态。
