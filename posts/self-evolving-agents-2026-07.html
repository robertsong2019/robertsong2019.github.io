当 AI Agent 学会自我进化：从反思到自我重写的进化之路

📅 2026-07-08 · 标签：AI Agent · 自我进化 · 技能库 · 强化学习 · LLM

想象你雇了一个助手，第一天表现还不错。但三个月后，你发现他还在用完全相同的方式工作——同样的错误，同样的效率，没有任何改进。你会怎么想？

这就是今天绝大多数 AI Agent 的现状。它们在第一千次执行任务时的表现，和第一次完全一样。没有积累，没有反思，没有成长。

但变化正在发生。从 2023 年的 Reflexion 到 2026 年的 Darwin Gödel Machine，一类全新的 Agent 正在崛起——它们能从自己的经验中学习，积累可复用的技能，甚至重写自己的代码。这不是微调，不是重新训练，而是一种更根本的能力：自我进化。

一、为什么"冻结的"Agent 注定不够用

今天的 LLM Agent 有一个根本性的架构缺陷：部署即冻结。模型权重固定不变，工作流一旦设定就不再调整，每次对话结束后一切归零。

这带来的问题很实际：

  • 重复犯错——同一个类型的任务，第 100 次和第 1 次的失败率一样高
  • 无法复用经验——解决过一次的复杂问题，下次还得从头来
  • 无法适应环境变化——API 改了、工具更新了、业务逻辑变了，Agent 浑然不知

一个统计数据很能说明问题：85% 的 AI 项目未能交付预期商业价值，主要原因不是模型不够强，而是系统部署后无法进化。

斯坦福大学在 2025 年秋季专门开设了 CS329A 课程《Self-Improving AI Agents》，由 Aakanksha Chowdhery 和 Azalia Mirhoseini 主讲。这标志着"Agent 自我进化"已经从边缘实验走向主流学术议题。

二、自我进化的四个层级

自我进化不是一个开关，而是一个从浅到深的频谱。理解这个层级结构，是设计和选型 Agent 系统的基础。

层级 名称 机制 代表系统 改进范围
L0 Prompt 级 同一对话内自我批评和修正 Reflexion, Self-Refine 单次会话
L1 技能级 跨会话积累可复用技能 Voyager, Hermes, SAGE 持久，可审计
L2 架构级 修改自身工作流和工具链 Darwin Gödel Machine 系统层面
L3 权重级 直接更新模型参数 SWE-RL, Agent-R1 根本性改变

一个简单的类比：L0 就像你在做数学题时发现思路错了，擦掉重写；L1 就像你整理了一个错题本，下次考试前翻一遍；L2 就像你觉得自己的学习方法不对，换了套新的学习策略；L3 则相当于你做了个脑部手术，直接改了神经突触。

每一层的天花板更高，但风险也更大。目前工业实践集中在 L1（技能级），因为它在效果和安全性之间取得了最佳平衡。

三、技能库：Agent 的"错题本"

几乎所有成功的自进化系统都有一个核心组件：技能库。

这个概念最早由 Voyager（NVIDIA/Caltech/Stanford, 2023）在 Minecraft 环境中验证。Voyager 不是一个普通的游戏 Bot——它能自动探索世界、采集资源、合成工具、甚至写代码控制角色。关键在于：每次成功完成一个任务，它都会把解决方案编码成一个命名良好的、可复用的 JavaScript 函数，存入技能库。遇到新任务时，先搜索库中有没有相关技能可以直接调用。

类比一下：这就像一个厨师的菜谱本。第一次做一道菜可能要试错很多次，但一旦成功了，菜谱就记下来。下次再做类似的菜，直接翻菜谱，不用从头摸索。

Amazon 的 SAGE（Skill Augmented GRPO for Self-Evolution, 2025-2026）将这个概念推向了新高度。它引入了 Sequential Rollout 机制：在一条相似任务链上依次部署 Agent，前序任务生成的技能自动累积，供后续任务复用。更重要的是，SAGE 用强化学习训练 Agent 学会如何创建和使用技能，而不只是被动存储。

结果令人印象深刻：

  • +8.9% 场景目标完成率提升
  • -59% 输出 token 减少
  • -26% 交互步骤减少
关键数据点：Token 减少几乎和准确率提升一样重要。这意味着随着技能积累，Agent 不只是更准确了，而是更高效了——用更少的计算资源解决同样复杂的问题。这就是复合效应：技能越多，效率越高，积累新技能的速度也越快。

2026 年 2 月的 SkillRL（ICLR 2026 Workshop）进一步发展了这个思路。它引入了 SkillBank——一个分层的技能库结构，配合递归进化机制，让技能库和 Agent 策略共同进化。在 ALFWorld 基准上达到了 89.9% 的完成率，而 GPT-4o 只有 75.3%。一个 7B 参数的开源模型，靠技能库的威力，打败了 GPT-4o。

四、反思循环：最朴素但也最危险的自我改进

最基础的自改进形式，是让 Agent 在推理时"反思"自己的行为。

Reflexion（Shinn et al., NeurIPS 2023）开创了这个方向。它的循环很简单：

  1. Agent 尝试解决任务
  2. 观察结果（成功/失败、错误信息）
  3. 用自然语言写一份"反思笔记"——哪里做错了？应该怎么改？
  4. 将反思存入记忆
  5. 下次尝试时，带着之前的反思一起

在 HumanEval 编程基准上，Reflexion 将 GPT-4 的通过率从 80% 提升到了 91%。这被称为"语言强化学习"（Verbal RL）——不需要更新任何权重，纯粹靠自然语言的自我批评来改进。

Self-Refine（Madaan et al., NeurIPS 2023）更加简洁：同一个 LLM 充当生成器、评估器和精炼器。先生成一个初始答案，然后自我评价，再根据反馈精炼。循环往复。整个过程不需要任何训练数据或强化学习。

但这里有一个陷阱。2024 年的批判性综述（Huang et al.）揭示了一个残酷现实：没有外部反馈信号的纯自我反思，效果往往不稳定甚至为负。原因很简单——LLM 在没有外部锚点的情况下，可能在"自我强化"而非"自我纠正"。就像一个学生自己批改自己的作业，越改越自信，但不一定越改越对。

真正有效的反思需要外部信号：

  • 代码执行结果（编译通过没？测试过了没？）
  • 工具调用的返回值（API 调用成功了吗？）
  • 搜索引擎的验证（事实是否正确？）
  • 多 Agent 之间的辩论（不同视角的碰撞）

五、开放式探索：进化的秘密武器

到目前，我们讨论的系统都有一个隐含限制：它们只从"最新版本"出发来改进。就像一个运动员只和自己比——永远在同一基础上优化。

Darwin Gödel Machine（Sakana AI, ICLR 2026）打破了这个限制。它的名字本身就说明了野心：结合达尔文进化和哥德尔机（一种理论上的自我改进 AI）。

DGM 的核心创新是开放式探索 + Agent 档案库：

  1. 维护一个不断增长的 Agent 档案库
  2. 从档案中采样一个 Agent（不是总选最新的）
  3. 让这个 Agent 修改自己的 Python 代码
  4. 在基准测试上验证修改效果
  5. 如果改进了，新 Agent 加入档案

这就像生物进化保持基因多样性：偶尔"回到过去"从一个不同的起点重新探索，可能发现一直往前走时永远碰不到的新路径。

结果确实惊人：

  • SWE-bench：20.0% → 50.0%
  • Polyglot：14.2% → 30.7%

更令人惊讶的是 DGM 自动发现了什么：更好的代码编辑工具、长上下文管理策略、甚至多方案生成+排序的机制。这些不是人类工程师设计的，而是系统自主探索出来的。

核心发现：DGM 的消融实验表明，自我改进和开放式探索缺一不可。只有自我改进（总从最新版本出发）会陷入局部最优；只有开放式探索（没有系统性的改进方向）则效率太低。两者的结合才是关键。

六、多 Agent 共进化：从单打独斗到团队演化

当单个 Agent 的自我改进遇到瓶颈时，一类新方向出现了：多 Agent 共进化。

Multi-Agent Evolve（2025年10月）设计了一个精巧的系统：三个角色由同一个 LLM 扮演——

  • Proposer：生成有挑战性的问题
  • Solver：尝试解决
  • Judge：评估双方的表现

这三个角色形成一个闭环：Proposer 不断出更难的题，Solver 被迫变强，Judge 的评估标准也在进化。在 Qwen2.5-3B-Instruct 上，这个系统在无人工监督的情况下，跨多个基准获得了平均 4.54% 的提升。

2026年3月的 SAGE（Multi-Agent 版本）将其扩展为四个角色：Challenger、Planner、Solver、Critic，加入了课程漂移预防机制。在 LiveCodeBench 上 +8.9%，OlympiadBench 上 +10.7%。

为什么多 Agent 比单 Agent 更有效？核心原因在于反馈信号的多样性。一个 Agent 自我评估有盲点（就像你很难发现自己的思维漏洞），但不同角色的 Agent 可以提供互补的视角。Proposer 知道什么问题难，Solver 知道什么策略有效，Judge 知道什么答案对。

七、生态与标准化：从实验室到产品

2025 年 12 月，Anthropic 发布了 Agent Skills 开放标准。这不是一个技术突破，而是一个生态里程碑——Microsoft、OpenAI、Atlassian、Figma、Cursor 和 GitHub 先后采用。

为什么这很重要？因为技能库的价值取决于可互换性。如果每个 Agent 系统的技能格式都不同，技能就无法跨系统复用。Agent Skills 标准做的事情，相当于为 Agent 技能定义了一个"USB 接口"——任何 Agent 都可以创建，任何 Agent 都可以使用。

2026 年 2 月，NousResearch 发布了 Hermes Agent——一个 MIT 协议的开源自改进 Agent。8700+ GitHub stars，142 个贡献者。它的工作方式很直观：

  1. 完成复杂任务后（通常涉及 5+ 次工具调用），自动生成一份结构化的 Markdown 技能文档
  2. 文档包含：步骤、常见陷阱、验证方法
  3. 遇到类似任务时，自动检索和更新技能

这正是 OpenClaw 等 Agent 框架正在采用的 Skill 模式——人类可读、可审计、可共享。

九、自我进化的阴暗面

自我进化的潜力令人兴奋，但也不能忽视风险。

1. 反馈摩擦（Feedback Friction）

2025 年的研究（Jiang et al.）揭示了一个叫做"Feedback Friction"的现象：LLM 往往无法完全吸收外部反馈。即使给了正确的纠正，模型也可能只接受一部分，或者曲解反馈的含义。这意味着自我改进的效率存在固有限制。

2. 技能库膨胀

Voyager 的技能库只增不减。随着库的增长，检索效率下降，噪音增加。如何淘汰过时技能？如何处理冲突的技能？Memory-R1（2025）用 RL 训练了一个专门的 Memory Manager，学习 ADD、UPDATE、DELETE 操作，但这本身就是一个复杂的优化问题。

3. 安全悖论

Agent 越能自我改进，就越难审计。L1（技能库）是安全的——技能是人类可读的 Markdown 文档，可以逐条审查。L2（架构修改）就危险多了——Agent 修改了自己的代码后，你真的能理解每一处改动的意图吗？L3（权重更新）几乎是个黑箱。

实践建议：如果你在构建自进化 Agent 系统，优先选择 L1（技能库）方案。它提供了最好的效果/安全比。确保所有自动生成的技能可以被人类审查，并且有版本控制。永远不要让 Agent 在没有人类监督的情况下修改自己的核心架构。

十、你现在就可以做什么

说了这么多研究，如果你是一个工程师或团队领导，今天能做什么？

立即可做（1-2周）

  1. 加入反思循环：在你的 Agent 系统中，每次任务失败后，让 Agent 写一段简短的"教训笔记"，保存到持久化存储中。下次类似任务时，检索相关笔记作为上下文。这不需要任何模型训练，只需要 prompt 设计和存储层。
  2. 实现基础技能库：当 Agent 成功完成一个复杂任务（比如 5+ 步骤的工具调用链），让它自动生成一份 SKILL.md 文件，包含步骤、注意事项、验证方法。保存到文件系统中，下次优先检索。

短期实验（1-3月）

  1. 设计技能验证流水线：不是所有自动生成的技能都是好技能。需要一个"测试-验证-保留/丢弃"的机制。对代码类技能，可以用单元测试验证。对策略类技能，可以用 A/B 测试。
  2. 实验 Sequential Rollout：找一组相似的任务，让 Agent 依次处理。测量技能复用的效果：第二次比第一次快多少？Token 节省了多少？准确率提升了多少？

中期方向（3-6月）

  1. 多 Agent 互相评估：部署两个不同角色的 Agent——一个执行，一个审查。审查 Agent 的反馈作为执行 Agent 的改进信号。这比单 Agent 自我评估更可靠。
  2. 关注 Agent-R1 方向：将 Agent 的每一步交互（而非整个对话）视为独立的强化学习转移。这在长链任务中特别有价值——可以对关键步骤做精确的奖励/惩罚，而不是笼统地评价整个任务。

结语

从 2023 年 Voyager 在 Minecraft 里积攒技能，到 2026 年 Darwin Gödel Machine 在 SWE-bench 上自我进化到 50%，自改进 Agent 的进步速度令人瞩目。

但最重要的趋势不是某个具体系统的突破，而是一个范式的转变：评价一个 Agent 的标准，正在从"它能做什么"变成"它能不能变得更好"。

就像招聘时，你不会只看候选人现在的技能，还会看他们的学习能力。Agent 也是如此。一个今天还不够完美但能持续进化的 Agent，比一个今天很强但永远不会变的 Agent，更有长期价值。

2026 年选择 AI Agent 的核心问题不再是"它能完成这个任务吗"，而是"它明天能比今天做得更好吗？"

能回答"是"的 Agent，才值得长期投入。
