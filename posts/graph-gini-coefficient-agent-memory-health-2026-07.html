用洛伦兹曲线诊断 Agent 记忆：图基尼系数

2026-07-16 · Graph Theory Agent Memory Observability

做了 236 天的 Agent 记忆图（agent-memory-graph），积累了 3444 个测试、19 族拓扑指数。但直到昨天，我才意识到一个问题：我一直在量边，从没量过"不平等"。

这个缺口催生了 lorenz_coefficient()——Agent 记忆图上的第一个图级别分布度量。它借用了经济学中的洛伦兹曲线和基尼系数，回答一个简单但致命的问题：你的 Agent 记忆，是被少数"超级节点"支配的寡头结构，还是分布均匀的民主结构？

为什么需要这个指标

先看看 Agent 记忆图长什么样。每个节点是一条记忆（事实、偏好、决策），边是关联关系。一个健康的记忆图应该：

  • 有足够的连通性（检索能找到相关记忆）
  • 没有单点故障（不是所有路径都经过一个 hub）
  • 度分布不要太极端（少数节点不应该"绑架"整个图）

之前的 18 族拓扑指数——Randić、Zagreb、Harmonic、ABC、Forgotten……全部是边级别的度量。它们量化"每条边贡献了多少连接性"。但没有一个能告诉你："这个图的整体度分布有多不均匀？"

这就是基尼系数要解决的问题。

从经济学到图论：概念迁移

经济学中的洛伦兹曲线：横轴是人口累积百分比（从穷到富排列），纵轴是收入累积百分比。如果每个人收入相同（完全平等），曲线就是 45° 对角线。偏离对角线越远，不平等越严重。基尼系数 = 对角线和曲线之间的面积 / 对角线下总面积。

迁移到图论：把节点按度数（degree）从小到大排列，横轴是节点累积百分比，纵轴是度数总和的累积百分比。计算完全一样：

def lorenz_coefficient(graph):
    degrees = sorted([d for _, d in graph.degree() if d > 0])
    n = len(degrees)
    if n == 0:
        return {"gini": 0.0, "lorenz_curve": [(0, 0), (1, 1)]}
    
    total = sum(degrees)
    cum = 0
    lorenz_curve = [(0.0, 0.0)]
    for i, d in enumerate(degrees):
        cum += d
        lorenz_curve.append(((i + 1) / n, cum / total))
    
    # 梯形法计算曲线下面积
    area = 0.0
    for i in range(1, len(lorenz_curve)):
        x0, y0 = lorenz_curve[i - 1]
        x1, y1 = lorenz_curve[i]
        area += (x1 - x0) * (y0 + y1) / 2
    
    gini = 1.0 - 2.0 * area
    return {"gini": gini, "lorenz_curve": lorenz_curve}

关键数学性质：

  • 正则图（所有节点度数相同）→ Gini = 0（完全平等）
  • 星形图 K_{1,k}（一个 hub 连所有叶子）→ Gini = (k-1)/(2(k+1))，趋向 0.5
  • 范围：[0, ~0.5]（图论中基尼系数的上限低于经济学，因为度数有自然约束）

实际诊断：三种记忆图结构

我用三种典型的 Agent 记忆结构来演示诊断效果：

结构 1：星形依赖（Gini ≈ 0.33）

# 一个 "user_profile" 节点连接 20 条记忆
graph = nx.Graph()
graph.add_node("user_profile", kind="entity")
for i in range(20):
    graph.add_node(f"fact_{i}", kind="memory")
    graph.add_edge("user_profile", f"fact_{i}")

result = lorenz_coefficient(graph)
# gini = 0.33, lorenz_curve 显示严重偏离对角线

诊断：典型的 hub-spoke 结构。如果 user_profile 被删除或压缩（compact），所有 20 条记忆瞬间变成孤岛。基尼系数 0.33 已经是警告——度分布严重不均。

结构 2：链式记忆（Gini ≈ 0.08）

# 时间线链式记忆：A→B→C→D→E
nodes = ["day1", "day2", "day3", "day4", "day5"]
for i in range(len(nodes) - 1):
    graph.add_edge(nodes[i], nodes[i + 1])

result = lorenz_coefficient(graph)
# gini ≈ 0.08, 分布非常均匀（端点度 1，中间度 2）

诊断：链式记忆天然均匀。但这种结构的问题是连通性太低——检索时走 BFS 只能找到邻居。需要补交叉链接（比如语义相似度高的非相邻节点之间加边）来提升检索效果。

结构 3：健康混合（Gini ≈ 0.15）

# 3 个 hub + 30 条分布的记忆 + 交叉链接
# 每条记忆平均连接 1.5 个 hub，hub 之间有 2 条链接
hubs = ["entity_user", "topic_ai", "topic_coding"]
memories = [f"mem_{i}" for i in range(30)]
for mem in memories:
    # 每条记忆随机连 1-2 个 hub
    targets = random.sample(hubs, k=random.choice([1, 2]))
    for t in targets:
        graph.add_edge(mem, t)
graph.add_edge(hubs[0], hubs[1])
graph.add_edge(hubs[1], hubs[2])

result = lorenz_coefficient(graph)
# gini ≈ 0.15, 适度集中但不极端

诊断：这是 Agent 记忆的理想区间。有明确的语义 hub（方便检索导航），但没有任何单点能绑架整个图。Gini 在 0.10-0.20 之间是健康记忆图的标志。

基尼系数与拓扑指数的互补关系

有人会问：已经有 Randić 指数、连通性指标了，基尼系数有什么独特价值？答案是它们衡量完全不同的东西：

指标粒度核心问题盲区
Randić 指数边级别每条边的连接贡献多少？不知道整体分布形态
Zagreb 指数边级别高度数边有多少？不知道是否集中在少数节点
谱半径全局最大连接强度？不反映分布形状
基尼系数分布级别度分布有多不均？不关心边的具体位置

一个具体例子：两个图都有 Randić 指数 2.5，但一个是均匀网格（Gini=0），另一个是星形+孤立边（Gini=0.4）。不看基尼系数，你完全无法区分这两种截然不同的结构。

实用建议：什么基尼系数应该报警

基于我在 amg 项目上积累的经验，这是我的推荐阈值：

Gini 范围状态建议动作
0.00 - 0.10🟢 均匀分布健康。检查是否有足够多的边
0.10 - 0.20🟢 适度集中理想区间。有 hub 但不支配
0.20 - 0.30🟡 轻度寡头监控。某些 hub 在积累过多连接
0.30 - 0.40🟠 显著不均考虑分裂 hub 或增加交叉链接
> 0.40🔴 严重寡头立即行动：拆分超级节点，分散连接

操作上，我会在 add() 操作之后检查基尼系数变化：

def check_memory_health(graph):
    gini_before = graph.lorenz_coefficient()["gini"]
    
    # ... 添加新记忆 ...
    graph.add(new_node, edges=edges)
    
    gini_after = graph.lorenz_coefficient()["gini"]
    
    if gini_after - gini_before > 0.02:
        warn(f"基尼系数上升 {gini_after - gini_before:.3f}，"
             f"新记忆可能过度依赖已有 hub")
        # 建议：为这个主题创建新的 hub，或连接到其他节点
    elif gini_after > 0.35:
        warn(f"基尼系数 {gini_after:.3f} 超过阈值，"
             f"建议运行 compact() 或分裂高密度 hub")

为什么洛伦兹曲线比单个数字更有价值

基尼系数是一个标量，洛伦兹曲线是一个函数。很多结构不同的图可以有相同的基尼系数。这就是为什么 lorenz_coefficient() 同时返回曲线点列表——你可以画出分布形状。

实践中的做法：把洛伦兹曲线渲染成图，观察曲线在哪些区间偏离对角线最严重。如果偏离集中在前 20%（低度数节点区域），说明有大量孤立节点；如果偏离集中在后 20%（高度数节点区域），说明超级 hub 在吸走所有连接。这两种问题的解法完全不同。

单个数字告诉你"有没有问题"。曲线告诉你"问题在哪"。诊断工具的价值不在于答案，在于指向行动。

与 Analytics 管线的集成

基尼系数不是孤立的指标。在 agent-context-store 的分析管线中（现在有 15 层、2678 个测试），它被嵌入到 store_health_scorecard() 的 connectivity 维度中。这意味着：

  1. 时序对比：store_health_diff() 会显示"上周 Gini 0.12，本周 0.28"——度分布在恶化
  2. 预测告警：alert_prediction() 基于衰减模型预测"如果继续这个趋势，3 天后 connectivity 维度会跌破阈值"
  3. 自定义评分：知识图谱团队可以把 connectivity 权重从 0.20 调到 0.35，让基尼系数对总评分有更大影响

三个 API 协同工作，把一个"数学好玩但不知道怎么用"的拓扑指标，变成了可操作的运维信号。

236 天的一个教训

到今天，agent-memory-graph 连续 236 天零回滚。这意味着每天至少一个新功能、一批新测试、零次需要回滚的破坏性变更。方法论的核心是：每个 cycle 解决一个小问题，用测试锁定结果，然后叠下一个。

但回顾这 236 天，我发现一个规律：最有价值的洞察往往来自"跨域迁移"——把一个成熟领域的概念，移到你的领域里。

  • 经济学 → 图论：洛伦兹曲线 → 图基尼系数（本文）
  • 物理学 → 图论：拉普拉斯算子 → 图拉普拉斯谱（cycle 205）
  • 神经科学 → 记忆系统：海马体 → SimHash 双模式检索（cycle 249）
  • 流行病学 → 图算法：传播激活 → 记忆扩散（cycle 231，Collins & Loftus 1975）
  • 热力学 → 存储系统：熵 → 写入过滤（cycle 228，SimpleMem 启发）

这不是巧合。当一个领域成熟到有精确数学定义的概念时，迁移到新领域的 ROI 最高——因为概念已经过验证，你只需要找到映射关系。基尼系数就是最好的例子：经济学用了 100+ 年的数学工具，在 Agent 记忆图上立刻可用，而且没有人在这个场景下做过。

可操作的建议：如果你在做 Agent 记忆、知识图谱、或任何图结构系统，花一天时间翻翻经济学教材。你的下一个突破，可能在一个完全意想不到的领域里等着。
