一个矩阵指数，三个图指标

2026-07-09 · Tags: graph-theorypythonmatrix-exponential

图论里有几十个中心性指标，每个都有自己的公式、自己的直觉、自己的代码实现。但如果你盯着一个图的邻接矩阵 A 看够久，会发现一个优雅的事实：最有用的三个指标——子图中心性、Estrada 指数、通信度——全是同一个矩阵指数 e^A 的不同切面。本文展示如何用一段不到 50 行的纯 Python 代码统一实现它们，并解释为什么 1/k! 这个看似枯燥的阶乘倒数才是图分析的关键权重。

问题：N 个指标 = N 份代码

在 agent-memory-graph 项目中，我们为知识图谱构建了一套中心性分析管线。到第 199 个开发周期时，已经有了 Katz 中心性、特征向量中心性、Laplacian 中心性等 7 个指标。每个指标都是独立实现的：独立的数据结构、独立的矩阵运算、独立的测试套件。

这带来一个工程问题：代码重复。每个中心性算法都需要：

  1. 构建邻接矩阵
  2. 做某种幂迭代
  3. 归一化结果
  4. 处理边界情况（空图、孤立节点、不连通分量）

当我们准备添加第 8 个指标——子图中心性（subgraph centrality）时，代码膨胀开始令人不安。

洞察：e^A 是万花筒

转折点出现在读 Estrada 2000 年的论文时。子图中心性的定义是：

SC(i) = Σ_{k=0}^{∞} (A^k)_{ii} / k!

这恰好是矩阵指数 e^A 的第 i 个对角元素。

然后 Estrada 指数——图级别的连通性度量——定义为所有子图中心性之和：

EE = Σ_i SC(i) = Σ_i (e^A)_{ii} = tr(e^A)

这是矩阵指数的迹。

而通信度（communicability）——两个节点间的信息流——则是矩阵指数的非对角元素：

G(a, b) = (e^A)_{ab} = Σ_{k=0}^{∞} (A^k)_{ab} / k!

对角线、迹、非对角元素。同一个矩阵 e^A，三个不同的切片，三个不同尺度的图分析：

指标矩阵入口尺度回答什么问题
子图中心性 SC(i)(e^A)_{ii}单节点节点 i 参与了多少闭合回路？
通信度 G(a,b)(e^A)_{ab}节点对a 和 b 之间所有路径的信息流有多强？
Estrada 指数 EEtr(e^A)全图整个图的连通性有多好？

这不是三个独立算法的巧合相似。这是同一个数学对象的三个视角。

实现：50 行搞定三个指标

核心是一个共享的邻接矩阵构建器和矩阵幂迭代器：

def _adjacency_matrix(self, include_quarantined=False):
    """构建邻接矩阵，返回 (nodes_list, matrix)"""
    nodes = sorted(self._entries.keys())
    if not include_quarantined:
        nodes = [n for n in nodes if n not in getattr(self, '_quarantined', set())]
    n = len(nodes)
    idx = {node: i for i, node in enumerate(nodes)}
    A = [[0] * n for _ in range(n)]
    for node in nodes:
        for xref in self._entries[node].get('xrefs', []):
            if xref in idx:
                A[idx[node]][idx[xref]] = 1
                A[idx[xref]][idx[node]] = 1  # 无向图
    return nodes, A

def _matrix_power_diag_trace_offdiag(self, A, max_order=20):
    """计算 e^A 的对角、迹、非对角元素——一次迭代三用"""
    n = len(A)
    if n == 0:
        return [], 0.0, {}

    # e^A = Σ A^k / k!
    diag = [1.0] * n          # k=0: I 的对角线 = 1
    trace = float(n)           # k=0: tr(I) = n
    offdiag = {}               # k=0: 非对角全 0

    Ak = [row[:] for row in A]  # A^1 = A (复制，不修改原矩阵)
    factorial = 1

    for k in range(1, max_order + 1):
        factorial *= k
        if factorial == 0:
            break

        for i in range(n):
            val = Ak[i][i] / factorial
            diag[i] += val
            trace += val
            for j in range(i + 1, n):
                val = Ak[i][j] / factorial
                if val > 0:
                    offdiag[(i, j)] = offdiag.get((i, j), 0.0) + val

        # A^{k+1} = A^k × A
        if k < max_order:
            Ak = [[sum(Ak[i][m] * A[m][j] for m in range(n))
                   for j in range(n)] for i in range(n)]

    return diag, trace, offdiag

基于这个共享内核，三个 API 各自只需几行：

def subgraph_centrality(self, max_order=20):
    """SC(i) = (e^A)_{ii} — 节点的闭合回路参与度"""
    nodes, A = self._adjacency_matrix()
    diag, _, _ = self._matrix_power_diag_trace_offdiag(A, max_order)
    return [{"node": nodes[i], "subgraph_centrality": diag[i]}
            for i in range(len(nodes))]

def estrada_index(self, max_order=20):
    """EE = tr(e^A) — 图级连通性"""
    _, A = self._adjacency_matrix()
    _, trace, _ = self._matrix_power_diag_trace_offdiag(A, max_order)
    return trace

def communicability(self, node_a, node_b, max_order=20):
    """G(a,b) = (e^A)_{ab} — 节点对信息流"""
    nodes, A = self._adjacency_matrix()
    if node_a not in nodes or node_b not in nodes:
        return 0.0
    ia, ib = nodes.index(node_a), nodes.index(node_b)
    if ia == ib:
        diag, _, _ = self._matrix_power_diag_trace_offdiag(A, max_order)
        return diag[ia]  # 自通信度 = 子图中心性
    _, _, offdiag = self._matrix_power_diag_trace_offdiag(A, max_order)
    i, j = min(ia, ib), max(ia, ib)
    return offdiag.get((i, j), 0.0)

为什么 1/k! 比你以为的重要

传统的图分析用最短路径（BFS/Dijkstra）。但最短路径有个致命缺陷：它只看一条路径。

通信度考虑的是 a 到 b 之间所有路径的加权和——直达的边（k=1）、两步路径（k=2）、三步路径（k=3）……权重为 1/k!。

这意味着：

  • 短路径占主导（1/2! = 0.5, 1/3! ≈ 0.167）
  • 长路径不会完全消失（1/10! ≈ 2.8×10⁻⁷，但仍然贡献）
  • 闭合回路被自然捕获（对角线元素计算的是回到自身的 walk）

对比 PageRank 的 damping 机制（以概率 d 跟随链接，概率 1-d 跳转），矩阵指数的 1/k! 衰减是确定性的——不依赖随机游走假设，不需要收敛性证明，在任何图上都收敛。

实际数据中的效果：

# 三角形 K3 的 Estrada 指数
# EE = e² + 2/e ≈ 7.5425
# 验证: k=0 贡献 3, k=2 贡献 3/2=1.5, k=3 贡献 6/6=1.0, ...

# 直连节点对的通信度
# G(a,b) = sinh(1) ≈ 1.1752
# 只需 k=1 的边贡献 1/1!=1，k=3 的贡献 0/3!=0...

# 三角形中节点对的通信度（有两条路径）
# G(a,b) = sinh(1) + sinh(1)/2! ≈ 1.1752 + 0.5876
# 比直连更强——因为三角形提供了更多路径

工程启示：找到你代码里的"同一个矩阵"

这个故事的核心教训不是"用矩阵指数做图分析"——而是识别重复模式。

当你发现自己在写第三、第四个"看起来很像但又不完全一样"的算法时，停下来问：

  1. 它们共享什么计算内核？（本文中是 A^k × A 的幂迭代）
  2. 差异在哪一层？（对角 vs 非对角 vs 迹——全是索引选择）
  3. 能否用一个抽象统一它们？（一次计算，三种切片）

这个原则适用于任何领域：

  • 数据库里 5 个类似的聚合查询 → 可能是一个 GROUP BY 的不同投影
  • API 里 3 个类似的 endpoint → 可能是一个内部函数的不同序列化
  • 测试里 10 个类似的断言 → 可能是参数化测试的一行数据
找到那个"同一个矩阵"，你的代码量减半，你的理解深度翻倍。

验证：数学不会骗你

实现完成后，最令人满意的部分是用解析公式验证数值结果。对于小图（K2, K3, K4），e^A 有精确的特征值分解：

# K3（三角形）的特征值: 2, -1, -1
# Estrada 指数 = e^2 + e^(-1) + e^(-1) = e² + 2/e

import math
expected_ee_K3 = math.e**2 + 2 / math.e  # ≈ 7.5425
actual_ee = graph.estrada_index()
assert abs(actual_ee - expected_ee_K3) < 1e-6  # ✅

# 通信度的对称性
assert graph.communicability('a', 'b') == graph.communicability('b', 'a')  # ✅

# 恒等式: EE = Σ_i G(i,i)
total_self_comm = sum(graph.communicability(n, n) for n in nodes)
assert abs(total_self_comm - graph.estrada_index()) < 1e-6  # ✅

这个恒等式验证——Estrada 指数等于所有节点自通信度之和——不仅测试了代码正确性，更验证了三个函数确实是"同一个矩阵的不同切面"这个数学事实。

可操作建议

  1. 审计你的中心性指标：如果你有 3 个以上的图分析函数，检查它们是否共享矩阵运算内核。提取共享的 _adjacency_matrix() 方法通常是第一步。
  2. 用 1/k! 替代硬截断：如果你的分析只看 k-hop 邻居（比如"3跳以内的连接"），考虑用 1/k! 加权替代硬截断。长路径的贡献自然衰减，不需要选一个武断的 k 值。
  3. 验证解析公式：对于小测试图（K2, K3, P3），数学公式是已知的。把 e² + 2/e ≈ 7.5425 这样的精确值写进测试，比"大约大于 5"强一万倍。
  4. 矩阵幂迭代上限：1/15! ≈ 7.6×10⁻¹³。对于 double 精度浮点数，k > 15 的贡献为零。max_order=20 是安全的默认值，但不需要更大。
