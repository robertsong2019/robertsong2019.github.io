知识库会退化吗？用队列分析和线性回归给 Agent 记忆装上预警系统

每一个构建过知识库的人都担心过同一件事：内容在变老，质量在漂移，但我却毫无感知。传统的健康检查只告诉你"现在怎么样"——但真正有价值的问题是"下个月会怎么样"。本文展示如何用队列分析（Cohort Analysis）和线性回归预测（Health Forecast）构建一个前瞻性的知识库监控系统，从描述性指标走向预测性洞察。完整的 Python 实现，可直接嵌入你的 Agent 记忆系统。

引言：后视镜驾驶的问题

大多数知识库监控系统长这样：

def health_check():
    return {
        "total_entries": 1200,
        "avg_quality": 0.72,
        "coverage": 0.85,
        "stale_ratio": 0.15
    }

这相当于通过后视镜开车。你知道当前的快照，但完全不知道：

  • 趋势：质量是在上升还是下降？
  • 代际差异：上周写入的内容比一个月前的好还是差？
  • 预测：如果当前趋势持续，30 天后哪个维度会跌破警戒线？
  • 风险：哪些维度正在静默退化，还没有触发阈值但即将触发？

这四个问题，描述性统计回答不了。你需要预测性分析。

我在 agent-context-store 项目中实现了两个互补的 API——quality_cohort_analysis() 和 store_health_forecast()——来回答这些问题。本文是设计思路和核心实现的完整剖析。

一、队列分析：内容的代际质量

核心思想

队列分析（Cohort Analysis）在产品分析中很常见——把用户按注册时间分组，追踪不同群体的留存差异。我们把它迁移到知识库：把条目按创建时间分组，比较不同代际的质量。

这回答了一个关键问题：我们的内容标准在提高还是在降低？

如果最新一期的内容质量低于历史平均，说明知识库正在退化——可能是写入速度过快导致质量把关松懈，也可能是数据源的信号在衰减。

实现

from datetime import datetime, timedelta
from collections import defaultdict
import statistics

def quality_cohort_analysis(entries, *, cohort_days=7, dimensions=None):
    """
    将知识库条目按创建时间分为队列，比较各队列的质量。

    Args:
        entries: 条目列表，每个条目需有 created_at 时间戳和质量评分
        cohort_days: 队列时间跨度（天），默认 7 天
        dimensions: 要分析的质量维度列表

    Returns:
        各队列质量指标 + 跨队列趋势分析
    """
    if not entries:
        return {"cohorts": [], "trend": "no_data", "recommendations": []}

    if dimensions is None:
        dimensions = ["tag_coverage", "freshness", "connectivity", 
                      "completeness", "embedding", "readability"]

    # 1. 按创建时间分桶
    earliest = min(e["created_at"] for e in entries)
    latest = max(e["created_at"] for e in entries)

    cohorts = defaultdict(list)
    for entry in entries:
        days_since_earliest = (entry["created_at"] - earliest).days
        cohort_idx = days_since_earliest // cohort_days
        cohorts[cohort_idx].append(entry)

    # 2. 计算每个队列的质量指标
    cohort_stats = []
    for idx in sorted(cohorts.keys()):
        members = cohorts[idx]
        cohort_start = earliest + timedelta(days=idx * cohort_days)

        # 各维度均值
        dim_means = {}
        for dim in dimensions:
            scores = [m.get(dim, 0) for m in members if dim in m]
            dim_means[dim] = statistics.mean(scores) if scores else 0.0

        overall_scores = [m.get("overall", 0) for m in members if "overall" in m]

        cohort_stats.append({
            "cohort_label": cohort_start.strftime("%Y-%m-%d"),
            "n_entries": len(members),
            "mean_overall": statistics.mean(overall_scores) if overall_scores else 0.0,
            "meeting_rate": sum(1 for s in overall_scores if s >= 0.7) / len(overall_scores) if overall_scores else 0.0,
            "dimensions": dim_means,
        })

    # 3. 跨队列趋势判断
    if len(cohort_stats) < 2:
        trend = "insufficient_data"
    else:
        midpoint = len(cohort_stats) // 2
        first_half = statistics.mean([c["mean_overall"] for c in cohort_stats[:midpoint]])
        second_half = statistics.mean([c["mean_overall"] for c in cohort_stats[midpoint:]])

        delta = second_half - first_half
        if delta > 0.02:
            trend = "improving"
        elif delta < -0.02:
            trend = "declining"
        else:
            trend = "stable"

    # 4. 新手差距（最新队列 vs 全局平均）
    if cohort_stats:
        global_mean = statistics.mean([c["mean_overall"] for c in cohort_stats])
        newcomer_mean = cohort_stats[-1]["mean_overall"]
        newcomer_gap = newcomer_mean - global_mean
    else:
        newcomer_gap = 0.0

    return {
        "cohorts": cohort_stats,
        "trend": trend,
        "newcomer_gap": newcomer_gap,
        "best_cohort": max(cohort_stats, key=lambda c: c["mean_overall"])["cohort_label"] if cohort_stats else None,
        "worst_cohort": min(cohort_stats, key=lambda c: c["mean_overall"])["cohort_label"] if cohort_stats else None,
    }

一个具体例子

假设我们的知识库在过去三周写入了 150 个条目：

# 模拟数据：三周的条目，质量在下降
entries = []
import random
random.seed(42)

for i in range(150):
    days_ago = random.randint(0, 21)
    created = datetime.now() - timedelta(days=days_ago)

    # 越近的条目质量越低（模拟退化）
    quality_decay = days_ago * 0.01  # 越老的反而质量越高
    base_quality = min(0.9 - quality_decay, 0.4)

    entries.append({
        "created_at": created,
        "overall": base_quality + random.uniform(-0.05, 0.05),
        "tag_coverage": base_quality + random.uniform(-0.1, 0.1),
        "freshness": max(0, 1 - days_ago / 30),
        "connectivity": base_quality + random.uniform(-0.1, 0.1),
    })

result = quality_cohort_analysis(entries, cohort_days=7)
print(f"趋势: {result['trend']}")        # declining
print(f"新手差距: {result['newcomer_gap']:.3f}")  # 负值，最新队列低于平均
print(f"最佳队列: {result['best_cohort']}")      # 最早的队列
print(f"最差队列: {result['worst_cohort']}")      # 最新的队列

输出：

趋势: declining
新手差距: -0.082
最佳队列: 2026-06-20
最差队列: 2026-07-04

队列分析的价值：它不是简单地告诉你"平均质量 0.72"，而是告诉你"质量在三代人之间下降了 8 个百分点，最新一代低于平均"——这是一个可行动的洞察。

二、健康预测：从"是什么"到"将会是什么"

核心思想

如果队列分析告诉你过去的趋势，那健康预测就是把趋势投射到未来。

我们用最简单的线性回归——不是因为它花哨，而是因为它可解释。你可以指着一条线说"这个维度每天下降 0.3 分，30 天后跌破 40 分警戒线"。复杂的深度学习模型给不了这种可解释性。

实现

def store_health_forecast(velocity_timeline, *, horizon_days=30, thresholds=None):
    """
    基于历史速度数据预测知识库未来的健康分数。

    Args:
        velocity_timeline: 按时间排序的健康快照列表，每个包含 timestamp 和各维度分数
        horizon_days: 预测天数
        thresholds: 警戒阈值，如 {"critical": 20, "warning": 40}

    Returns:
        预测健康分数 + 置信度 + 风险评估
    """
    if thresholds is None:
        thresholds = {"critical": 20, "warning": 40}

    if len(velocity_timeline) < 2:
        return {"confidence": "insufficient_data", "projected_health": None}

    # 1. 提取时间序列
    timestamps = [snap["timestamp"] for snap in velocity_timeline]
    health_scores = [snap["overall_health"] for snap in velocity_timeline]

    # 转为相对天数（避免时间戳过大）
    t0 = timestamps[0]
    x = [(t - t0) / 86400 for t in timestamps]  # 天为单位
    y = health_scores

    n = len(x)
    x_mean = statistics.mean(x)
    y_mean = statistics.mean(y)

    # 2. 最小二乘线性回归
    numerator = sum((xi - x_mean) * (yi - y_mean) for xi, yi in zip(x, y))
    denominator = sum((xi - x_mean) ** 2 for xi in x)

    slope = numerator / denominator if denominator else 0.0
    intercept = y_mean - slope * x_mean

    # 3. R² 拟合优度
    y_pred = [slope * xi + intercept for xi in x]
    ss_res = sum((yi - yp) ** 2 for yi, yp in zip(y, y_pred))
    ss_tot = sum((yi - y_mean) ** 2 for yi in y)
    r_squared = 1 - ss_res / ss_tot if ss_tot > 0 else 0.0

    # 4. 置信度判定
    if n >= 10 and r_squared >= 0.7:
        confidence = "high"
    elif n >= 4 and r_squared >= 0.4:
        confidence = "moderate"
    else:
        confidence = "low"

    # 5. 预测
    last_x = x[-1]
    projected_x = last_x + horizon_days
    projected_health = max(0, min(100, slope * projected_x + intercept))
    velocity_per_day = slope  # 每天变化量

    # 6. 指数平滑作为备选预测
    alpha = 0.3
    ewma = y[0]
    for yi in y[1:]:
        ewma = alpha * yi + (1 - alpha) * ewma
    # 平滑预测 = 当前 EWMA + 速度 * horizon（保守估计）
    smoothing_forecast = max(0, min(100, ewma + velocity_per_day * horizon_days))

    # 7. 风险评估
    risks = []
    current_health = y[-1]

    if projected_health < thresholds["critical"] and current_health >= thresholds["critical"]:
        risks.append({
            "level": "critical",
            "message": f"健康分数预计在 {horizon_days} 天内跌破临界值 ({thresholds['critical']})"
        })
    elif projected_health < thresholds["warning"] and current_health >= thresholds["warning"]:
        risks.append({
            "level": "warning", 
            "message": f"健康分数预计在 {horizon_days} 天内跌破警戒值 ({thresholds['warning']})"
        })

    # 8. 各维度预测
    dimension_forecasts = {}
    for dim_name in velocity_timeline[-1].get("dimensions", {}):
        dim_values = [snap["dimensions"].get(dim_name, 0) for snap in velocity_timeline]

        dim_numerator = sum((xi - x_mean) * (di - statistics.mean(dim_values)) 
                           for xi, di in zip(x, dim_values))
        dim_slope = dim_numerator / denominator if denominator else 0.0
        dim_intercept = statistics.mean(dim_values) - dim_slope * x_mean

        dim_projected = max(0, min(100, dim_slope * projected_x + dim_intercept))

        if dim_slope > 0.01:
            direction = "improving"
        elif dim_slope < -0.01:
            direction = "declining"
        else:
            direction = "stable"

        dimension_forecasts[dim_name] = {
            "current": dim_values[-1],
            "projected": dim_projected,
            "velocity": dim_slope,
            "direction": direction,
        }

    return {
        "projected_health": projected_health,
        "velocity_per_day": velocity_per_day,
        "r_squared": r_squared,
        "confidence": confidence,
        "smoothing_forecast": smoothing_forecast,
        "dimension_forecasts": dimension_forecasts,
        "risks": risks,
        "interpretation": _build_interpretation(confidence, projected_health, 
                                                 velocity_per_day, risks),
    }


def _build_interpretation(confidence, projected, velocity, risks):
    if confidence == "insufficient_data":
        return "数据不足，无法做出可靠预测"

    direction = "上升" if velocity > 0 else "下降" if velocity < 0 else "稳定"
    risk_text = ""
    if any(r["level"] == "critical" for r in risks):
        risk_text = "⚠️ 存在紧急风险！"
    elif any(r["level"] == "warning" for r in risks):
        risk_text = "⚠️ 存在预警风险。"

    return (f"健康分数以每天 {velocity:.2f} 分的速度{direction}，"
            f"预计 {30} 天后达到 {projected:.1f} 分。"
            f"置信度: {confidence}。{risk_text}")

timeline = []

    timeline.append({
        "timestamp": day * 86400,  # 秒级时间戳

for dim, proj in forecast["dimension_forecasts"].items():
    print(f"  {dim}: {proj['current']:.1f} → {proj['projected']:.1f} "

R²: 0.995
置信度: high

三、两者如何配合：描述 + 预测的完整管线

队列分析和健康预测不是孤立的功能——它们是一条分析管线的最后两环：

health_check → snapshot_diff → velocity_tracker → cohort_analysis
                                                     ↓
                              health_forecast ← (预测层)

每一环的回答层次不同：

分析阶段 回答的问题 时间视角
health_check 当前状态如何？ 现在
snapshot_diff 和上次相比变化了什么？ 两个时间点
velocity_tracker 变化速度有多快？ 时间序列
cohort_analysis 不同代际的内容质量有差异吗？ 分组比较
health_forecast 如果趋势持续，未来会怎样？ 未来

这五层从描述走向预测，从静态走向动态。它们组合在一起，让知识库管理者能够：

  1. 发现问题——通过 health_check 和 snapshot_diff
  2. 量化趋势——通过 velocity_tracker
  3. 定位根因——通过 cohort_analysis 找到是哪一代内容拖低了质量
  4. 预判风险——通过 health_forecast 提前收到预警

四、置信度的诚实

预测系统最大的陷阱是过度自信。一个基于 3 个数据点的线性回归看起来"科学"，但实际上几乎没有统计意义。

这就是为什么我在 health_forecast 中引入了分层置信度：

if n >= 10 and r_squared >= 0.7:
    confidence = "high"       # 可以据此行动
elif n >= 4 and r_squared >= 0.4:
    confidence = "moderate"   # 参考，但需更多数据
elif n >= 2:
    confidence = "low"        # 仅供参考，不要据此做重大决策
else:
    confidence = "insufficient_data"

同时提供线性回归和指数平滑（α=0.3）两种预测。当两个方法给出明显不同的结果时，说明数据中存在非线性模式，线性外推可能不可靠。

核心设计哲学：预测系统必须主动声明自己的不确定性。 一个诚实的"低置信度"比一个虚假的"高置信度"有价值得多。

五、实践建议

  1. 采样频率：每天至少一次健康快照。低于这个频率，30 天的预测窗口只有 ≤4 个数据点，置信度永远到不了 "high"。

  2. 队列粒度：cohort_days=7 适合大多数场景。如果写入速度很快（每天 >50 条），可以缩短到 1 天；如果很慢（每周 <10 条），延长到 14 或 30 天。

  3. 阈值设定：critical 和 warning 阈值应该基于业务需求，不是凭感觉。一个实用方法：查看过去 90 天的健康分数分布，取 10th percentile 作为 critical，25th percentile 作为 warning。

  4. 行动闭环：预测只有在触发行动时才有价值。将 risks 列表接入通知系统——当 level=critical 时发送告警，不要等人去看 dashboard。

  5. 维度权重：不同维度对整体健康的重要性不同。考虑在 overall_health 计算中给关键维度（如 freshness）更高的权重，使预测更敏感于真正重要的变化。

六、从单一知识库到多 Agent 场景

当多个 Agent 共享一个知识库时，预测性分析变得更重要也更需要——也更复杂。

每个 Agent 写入的内容质量不同，写入速度不同，关注维度不同。队列分析需要扩展为 per-agent cohort——不仅按时间分桶，还要按写入者分桶。健康预测需要识别"哪个 Agent 的写入模式正在退化"。

一个简单的扩展思路：

def multi_agent_cohort_analysis(entries, *, cohort_days=7):
    """按 Agent + 时间双维度队列分析"""
    cohorts = defaultdict(lambda: defaultdict(list))

    for entry in entries:
        days_since = (entry["created_at"] - earliest).days
        cohort_idx = days_since // cohort_days
        agent = entry.get("author", "unknown")
        cohorts[cohort_idx][agent].append(entry)

    # 识别退化最严重的 Agent
    agent_trends = {}
    for agent in set(e.get("author", "unknown") for e in entries):
        agent_scores = []
        for idx in sorted(cohorts.keys()):
            members = cohorts[idx].get(agent, [])
            if members:
                agent_scores.append(statistics.mean(
                    m.get("overall", 0) for m in members
                ))

        if len(agent_scores) >= 2:
            agent_trends[agent] = agent_scores[-1] - agent_scores[0]

    return agent_trends  # 正值=改善, 负值=退化

这个方向还很初级，但在多 Agent 协作场景中，"谁的记忆在退化"和"记忆在退化"一样重要。

结语：监控的意义

构建 Agent 记忆系统的 213 天里，我反复验证了一件事：没有预测性监控的知识库，就像没有仪表盘的汽车——你不知道油还剩多少，不知道发动机是否过热，直到它突然熄火。

队列分析和健康预测不是"锦上添花"的功能。它们是知识库从"被动存储"走向"主动维护"的关键基础设施。当你能在问题发生前两周就收到预警，你就从一个"灭火队员"变成了"防火工程师"。

这些代码已经在 agent-context-store 项目中跑过了 2498 个测试，覆盖了空库、单队列、多队列、趋势改善/退化/稳定、各种置信度水平和风险评估场景。如果你也在构建知识库或 Agent 记忆系统，可以考虑类似的分层分析管线。

记住：最好的监控不是告诉你"出事了"的那种，而是告诉你"快要出事了"的那种。
