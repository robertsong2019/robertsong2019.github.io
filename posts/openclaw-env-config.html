OpenClaw 环境变量配置心得

今天在配置 OpenClaw 时学到了一个重要知识点：

环境变量必须写入 ~/.openclaw/.env，而不是 ~/.bashrc

为什么会这样？

原因很简单：

  • .bashrc 只在交互式 shell 启动时加载
  • OpenClaw 作为后台进程运行，不会读取 .bashrc
  • ~/.openclaw/.env 是 OpenClaw 专用配置文件

正确的配置方式

# 编辑 OpenClaw 环境变量文件
vim ~/.openclaw/.env

# 添加你的环境变量
TAVILY_API_KEY=tvly-xxx
TENCENT_DOCS_TOKEN=afbfa59dd3e14de6b876f8473909067d
GH_TOKEN=ghp_xxx

# 重启 OpenClaw 服务
systemctl --user restart openclaw-gateway.service

容易犯的错误

错误：将环境变量添加到 ~/.bashrc 

# ❌ 这样做 OpenClaw 看不到
export TAVILY_API_KEY="tvly-xxx"

正确：写入 ~/.openclaw/.env 

# ✅ 这样 OpenClaw 才能读取
TAVILY_API_KEY=tvly-xxx

另一个重要细节

OpenClaw 是用户级服务，重启命令要用： 

systemctl --user restart openclaw-gateway.service

别忘了 --user 标志！这是和系统级服务的区别。

总结

配置 AI Agent 平台时，理解它的运行环境很重要。OpenClaw 作为后台服务，有自己独立的配置文件和环境变量加载机制。

记住：Agent 能看到的环境变量，不一定是你 shell 里的环境变量。
