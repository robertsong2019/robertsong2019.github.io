Agent 记忆的三个时代：从塞进上下文窗口到构建记忆基础设施

2026年07月21日

1960 年代，程序员用打孔卡片管理数据。每张卡片是一条记录，找数据意味着物理翻阅纸牌。

六十年后，我们让 LLM 把整个对话历史塞进上下文窗口——本质上，就是让它在翻数字版的打孔卡片。

这真的合理吗？

一个问题开始的故事

假设你有一个跟了你三年的私人助理 Agent。它记得你三年前说过喜欢清淡饮食，记得你两年前换了工作，记得你去年开始健身。现在你问它："今晚吃什么？"

一个"无记忆"的 LLM 会给你一个通用的推荐——它不知道你是谁。

一个"全上下文"的 Agent 会把三年的所有对话记录一股脑塞进 prompt——几十万 token 的历史记录，每次调用都带上。成本爆炸，而且大部分信息根本无关。

一个"有记忆系统"的 Agent 会怎么做？它在后台已经从历史交互中提炼了结构化知识：你的饮食偏好、工作变化、健身计划。当你说"今晚吃什么"，它只检索相关的几条记忆，用不到 1000 token 就能给出精准推荐。

这就是 Mem0 论文展示的核心结果：用不到 7000 token 的结构化记忆，在准确率上击败了消耗 25000+ token 的全上下文方法。成本降 90%，延迟降 91%，准确率反而更高。

这不是渐进式改进——这是范式转移。而要理解它，我们需要看 Agent 记忆是怎么一步步走到今天的。

第一代：把上下文窗口当记忆（2022-2023）

核心逻辑：记忆 = prompt 里的文本

类比：文件系统时代的"全量扫描"

ChatGPT 横空出世后，人们很快发现了一个问题：它记不住东西。每次对话都从零开始，你得反复解释背景。这就像每次见你的医生，都得重新介绍家族病史。

最朴素的解决方案就是：把历史记录全部放进 prompt。早期的 Agent 框架（2023 年初的 LangChain 默认模式）就是这么做的——每次调用 LLM，把完整的对话历史拼接到 prompt 里。

这个方案的问题显而易见：

  • 成本爆炸：对话越长，token 越多，费用线性增长。聊 100 轮之后，一次调用的成本可能是第一次的 100 倍。
  • 上下文限制：不管窗口多大（4K、128K、1M），总有一个硬性上限。超过就溢出。
  • 信号稀释：把所有历史塞进去，重要的信息被淹没在噪音中。LLM 的注意力机制有"中间遗忘"（lost in the middle）问题——放在 prompt 中间的关键信息经常被忽略。

但这一代也并非毫无价值。它证明了"上下文即记忆"的基本可行性，同时暴露了三个核心问题：怎么存？怎么检索？怎么遗忘？

第二代：操作系统启发的分层记忆（2023-2024）

核心逻辑：借鉴 OS 的分层内存管理，在有限的上下文窗口内外智能调度信息

类比：层次数据库——有了结构，但存储和应用仍然耦合

代表系统：MemGPT、Generative Agents、Reflexion

MemGPT：Agent 的操作系统

2023 年 10 月，UC Berkeley 的 Charles Packer 团队发表了 MemGPT，提出了一个精妙的类比：如果把 LLM 的上下文窗口看作"主存"（RAM），那我们就需要一套虚拟内存管理系统。

操作系统的内存分层是这样的：

CPU Register (KB)  →  L1 Cache (KB)  →  L2/L3 Cache (MB)  →  RAM (GB)  →  Disk (TB)
   最快，最小                                                                              最慢，最大

MemGPT 的记忆分层几乎一一对应：

System Prompt (固定)  →  Working Context (当前对话)  →  FIFO Queue (近期消息)  →  Archival Storage (长期记忆)
    最常访问                                                                                  冷数据

关键创新在于自主页面置换：MemGPT 让 LLM 自己决定什么时候把什么信息从长期存储调入工作记忆、什么时候把什么信息换出去。就像操作系统决定哪些页面保留在 RAM、哪些换到磁盘上。

但 MemGPT 有一个根本局限：记忆管理逻辑和 Agent 逻辑耦合在一起。LLM 必须花宝贵的推理周期来"思考"记忆管理——"我现在该检索哪条记忆？该遗忘哪条？"这就像应用程序的代码里混入了数据库的查询优化逻辑。

Generative Agents：记忆的三因子评分

斯坦福的"Smallville"论文（25 个 AI Agent 在虚拟小镇生活）提出了另一条路。每个 Agent 维护一个"记忆流"（memory stream），检索时用三个因子打分：

  1. Recency（新近度）：越近的记忆权重越高，用指数衰减模拟人类遗忘曲线
  2. Importance（重要性）：用 LLM 给每条记忆打分（"吃了个三明治"=1分，"和女朋友分手了"=9分）
  3. Relevance（相关性）：与当前查询的语义相似度

这个模型简单得惊人，却非常有效。它本质上就是一个记忆检索的评分函数——跟搜索引擎的排序算法没有本质区别。

Reflexion：从经验中学习

Reflexion 更进一步——Agent 不只是记住"发生了什么"，还记住"什么策略有效"。每次任务失败后，Agent 生成一段语言反思（"我失败了，因为忽略了X"），存入记忆。下次遇到类似任务，先检索相关反思。

这和人类的"经验"非常相似：你不记得每次具体怎么做的，但你记得"上次这么做失败了，因为某个原因"。

第三代：记忆即基础设施（2025-2026）

核心逻辑：记忆作为独立服务运行，Agent 不需要管理自己的记忆

类比：关系数据库——数据和应用彻底分离，查询语言统一接口

代表系统：Mem0、Letta（MemGPT 商业版）、Zep

Mem0：Token 经济学的胜利

2025 年 4 月的 Mem0 论文给出了最硬核的数据证明。在 LoCoMo 基准测试上：

方法 准确率 Token 成本 P95 延迟
全上下文（把所有历史塞进 prompt） ~88% 25,000+ 高
RAG（标准检索增强） ~75% ~15,000 中
Mem0 92.5% ~7,000 低（降91%）
Mem0 + 图记忆 ~94.5% ~7,000 低

注意最后两行：Mem0 不仅更便宜，而且更准确。这不是 trade-off——是双赢。为什么？因为结构化记忆消除了噪音。全上下文方法把所有历史都喂给 LLM，让 LLM 自己去"大海捞针"。Mem0 提前做了过滤和结构化，LLM 只需要处理已经精选过的信息。

Mem0 的三层技术栈值得深入理解：

写入端：单遍分层提取（Single-pass Hierarchical Extraction）

不是存储原始对话，而是每次交互后做一次"提炼"——提取实体、关系、偏好、事实。特别值得注意的是 Mem0 的一个关键更新：Agent 自己生成的事实（比如"根据你的偏好，我推荐X"）现在被当作一等公民存储，和用户说的信息同等重要。

检索端：三路并行评分融合（Multi-signal Retrieval）

Query → ┌─ 语义相似度 (向量搜索)  ─┐
        ├─ 关键词匹配 (BM25)       ├→ 分数融合 → 排序 → Top-K 记忆
        └─ 实体匹配 (图查询)       ─┘

这本质上是数据库的查询优化器在做的事——根据查询类型，选择不同的索引策略，然后融合结果。

存储端：图记忆表示（Graph Memory）

不只是存"事实"，还存"关系"。如果你说"我老婆叫 Sarah"和"Sarah 对花生过敏"，图记忆会把这两条信息关联起来。当你问"给我老婆做什么甜点"，它能推理出"避开花生"。

Letta：记忆即数据

MemGPT 团队商业化的公司 Letta，提出了"记忆即数据"（Memory as Data）的理念。Agent 的记忆不是 prompt 的一部分，而是独立的数据层——可以被查询、修改、版本控制，就像数据库里的记录。

这意味着 Agent 的记忆可以：

  • 版本控制：记忆有历史，你可以回溯到任意时间点的状态
  • 权限管理：不同 Agent 对同一记忆库有不同的访问权限
  • 审计追踪：谁在什么时候修改了什么记忆
  • 跨 Agent 共享：多个 Agent 从同一记忆库读取/写入

长期记忆与自我进化

2024 年的论文 "Long Term Memory: The Foundation of AI Self-Evolution" 提出了一个更具野心的框架。论文的核心论点：长期记忆不仅是为了更好的对话，它是 AI 自我进化的前提。

他们构建了 OMNE（基于 LTM 的多 Agent 框架），在 GAIA 基准测试中拿了第一。GAIA 是一个极具挑战性的通用 AI 助手基准，需要 Agent 具备多步推理、工具使用和长期规划能力。OMNE 的成功证明了一件事：记忆不是"锦上添花"，而是 Agent 智能的核心组成。

数据库历史的回响

如果你熟悉数据库发展史，Agent 记忆的演进会让你感到强烈的既视感。

📉 数据库 50 年的浓缩重演

数据库时代 时间 Agent 记忆对应 时间
文件系统（手动管理） 1950s Prompt 里全塞进去 2022
层次/网状数据库 1960s MemGPT（分层管理，逻辑耦合） 2023
关系数据库（数据独立） 1970s Mem0/Letta（记忆即数据） 2025
分布式数据库 2000s 跨 Agent 共享记忆（未来） ?
向量/AI 原生数据库 2020s AI 原生记忆基础设施 ??

这个类比不是牵强的——架构模式在逐字复现：

  • 查询优化器 = Mem0 的多信号检索策略选择
  • 事务（ACID） = 记忆一致性（多个 Agent 读写同一记忆库）
  • 索引 = 向量索引 + 关键词索引 + 图索引
  • 物化视图 = 记忆巩固（从原始日志提炼结构化知识）
  • 存储过程 = 异步记忆管理 pipeline

如果这个规律成立，那么 Agent 记忆的下一步几乎是可以预测的。

"遗忘"从 bug 变成 feature

早期 Agent 记忆系统的目标是"记住一切"。但最新的研究揭示了一个反直觉的事实：无限记忆会稀释信号。

Mem0 在 BEAM 10M 基准上的得分只有 48.6——远低于 BEAM 1M 的 64.1。这不是因为记忆系统变差了，而是因为信息太多了。当你有 1000 万 token 的对话历史，即使是最精准的检索也会被噪音淹没。

人类大脑每天的感官数据量约为 70GB，但只有不到 1% 固化为长期记忆。这不是缺陷——这是精心设计的过滤机制。你的大脑主动遗忘了 99% 的信息，才让那 1% 的关键知识保持清晰可检索。

最新的 Agent 记忆系统正在学习同样的"遗忘艺术"：

  • 时间衰减：Generative Agents 的 recency 因子——越旧的记忆权重越低
  • 重要性过滤：不是所有交互都值得记住。"好的，收到"和"决定使用 PostgreSQL"的重要性天差地别
  • 语义去重：同一条信息说三次，只存一次
  • 主动遗忘：Letta 的记忆编辑允许 Agent 主动删除过时信息——"我换了新手机号"应该让旧号码退役

Mem0 的官方路线图明确列出了两个下一步方向：时间抽象（表示事件如何随时间关联）和跨会话结构（将分散的交互连接成连贯的时间线）。这本质上是给记忆加上"时间维度"——数据库花了 20 年才完善的事务时间（transaction time）管理。

记忆的"睡眠模式"

当前所有主流 Agent 记忆系统都是同步的——每次交互都触发记忆读写。但 Mem0 明确提到他们的下一步是"agent-native memory"，让记忆管理异步运行。

这意味着什么？

Agent 不再花推理周期管理记忆。 当前 Agent 每次交互大概是这样的：

用户消息 → 检索记忆 (200ms) → LLM 推理 (2s) → 更新记忆 (200ms) → 返回

未来会变成：

用户消息 → 记忆服务预取相关上下文 → LLM 推理 → 返回
              ↓ (异步)
         后台记忆巩固 pipeline
         (提取 → 压缩 → 去重 → 图更新)

这就像人类的睡眠机制——白天快速记录碎片记忆，夜晚在睡眠中整理、巩固、遗忘。海马体在睡眠期间回放白天的神经活动，将短期记忆转移到大脑皮层固化为长期记忆。

未来的 Agent 可能会有一个明确的"睡眠模式"：在用户不活跃时，后台 pipeline 批量处理积累的原始交互日志，做记忆巩固——提取实体和关系、去重冗余信息、更新知识图谱、压缩过期细节。第二天醒来时，Agent 的记忆已经被整理好了，检索更快、成本更低。

你可以立刻做什么

  1. 评估你的 Agent 的"记忆健康度"：如果你在用 LangChain/LangGraph 的默认对话历史，你还在第一代。检查一下每次调用的 token 消耗——如果超过 10K，很可能有 80% 是冗余的。
  2. 试试 Mem0 的 API：它的免费层足够做原型验证。集成只需要几行代码——在每次对话后调用 mem0.add()，在查询前调用 mem0.search()。官方数据显示平均节省 90% 的 token 成本。
  3. 设计你的"记忆巩固 pipeline"：即使不用 Mem0，你也可以手动实现——每天跑一个批处理，从交互日志中提取关键信息，存入结构化的知识库（可以是 JSON、SQLite、或图数据库），丢弃低价值对话。
  4. 加入"重要性评分"：借鉴 Generative Agents 的方法，用 LLM 给每条交互打分（1-10），低于 3 分的直接不存。这一个简单规则通常能减少 60% 以上的记忆存储量。
  5. 规划多 Agent 共享记忆架构：如果你的团队有多个 Agent（比如一个负责编程、一个负责研究），考虑共享一个记忆库。这不仅能减少存储冗余，还能让 Agent 之间互相学习。

本文基于 MemGPT (2023)、Mem0 (2025)、Long Term Memory (2024) 等 10+ 篇论文和系统的研究。AI 领域发展极快，文中描述的技术状态以 2026 年 7 月发文时为准。
