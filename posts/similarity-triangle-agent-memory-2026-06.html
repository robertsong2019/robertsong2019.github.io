相似度三角：Agent 记忆系统的三维相似度工程

2026-06-17 · 罗嵩 · Agent Memory 相似度 信息论 TF-IDF

做 Agent 记忆系统的人都会撞到同一个问题：两条记忆"相似"到底意味着什么？是它们打了相同的标签？是它们用了相同的词汇？还是它们在语义空间里距离很近？这三件事经常被混为一谈，但它们是完全不同的维度。过去两周在 agent-context-store 项目里，我一步步把这三个维度拆开实现，得到一些不太显然的结论。

这篇文章记录这个"相似度三角"的完整构建过程——为什么需要三个维度，每个维度的工程实现长什么样，以及把它们组合起来后涌现出的能力。

                    ┌─────────────────┐
                    │  Similarity     │
                    │  Triangle       │
                    └────────┬────────┘
                   /         │         \
                  /          │          \
     tag_jaccard /    content_similarity  \  embedding_distance
      (分类维度)       (词汇维度)           (语义维度)
        J(A,B)        cosine(TF₁,TF₂)    cosine(emb₁,emb₂)

一、为什么单一维度不够

先看具体场景。Agent 记忆库里有这三条记录：

  • 记录 A：标签 [python, testing] — "How to mock datetime in pytest fixtures"
  • 记录 B：标签 [python, testing] — "pytest mark.skip behavior on parametrize"
  • 记录 C：标签 [java] — "Mocking static methods with Mockito framework"

用 标签 Jaccard（分类维度）看：A 和 B 相似度为 1.0（完全相同标签），A 和 C 为 0.0。但 A 和 C 在内容上显然更接近——都在讲 mock！

用 余弦词频相似度（词汇维度）看：A 和 C 的 "mock" 一词会让它们获得不错的分数。但 "pytest fixtures" 和 "Mockito framework" 的实际重叠词只有 "mock"（去停用词后），分数可能并不高。

用 Embedding 余弦距离（语义维度）看：A 和 C 会得到最高相似度，因为语义模型知道 pytest mock 和 Mockito mock 做的是同一件事。

结论很清楚：三个维度各有所长，单维度必然有盲区。标签维度会漏掉跨标签的相关性，词汇维度会被同义词困住，语义维度在短文本上不够精确。

二、第一维：分类相似度 — tag_jaccard

最简单也最容易被忽视的维度。实现就是经典的 Jaccard 系数：

def tag_jaccard(self, key1: str, key2: str) -> float | None:
    """两个条目标签集合的 Jaccard 相似度"""
    e1, e2 = self._get(key1), self._get(key2)
    if not e1 or not e2:
        return None
    t1, t2 = set(e1.tags), set(e2.tags)
    if not t1 and not t2:
        return 0.0
    intersection = t1 & t2
    union = t1 | t2
    return len(intersection) / len(union)

看起来朴素，但它解决了一个关键问题：标签是人类（或 Agent）显式标注的意图信号。当 Agent 给两条记忆打了同一个 "debugging" 标签时，这个行为本身就携带了信息——Agent 认为 "debugging" 对这两条记忆是核心属性。这种显式信号比任何隐式计算都更可信。

但标签维度有两个盲区：它无法发现跨标签的相关性（"pytest" 和 "unittest" 是相关的但标签不同），也无法衡量内容层面的重叠。这就是为什么需要第二维。

三、第二维：词汇相似度 — content_similarity

词汇维度用 TF 向量的余弦相似度。不需要 Embedding，不需要外部模型，纯靠词频统计：

import re
from collections import Counter
from math import sqrt

def content_similarity(self, key1: str, key2: str) -> float | None:
    """两条记忆内容的余弦词频相似度"""
    e1, e2 = self._get(key1), self._get(key2)
    if not e1 or not e2:
        return None
    if not e1.content or not e2.content:
        return 0.0

    def tf(text):
        words = re.findall(r'\w+', text.lower())
        return Counter(words)

    v1, v2 = tf(e1.content), tf(e2.content)
    if not v1 or not v2:
        return 0.0

    # 点积
    dot = sum(v1[w] * v2[w] for w in v1 if w in v2)
    # 模长
    mag1 = sqrt(sum(c * c for c in v1.values()))
    mag2 = sqrt(sum(c * c for c in v2.values()))

    if mag1 == 0 or mag2 == 0:
        return 0.0
    return dot / (mag1 * mag2)

这个实现有一个不太显然的设计决策：不做停用词过滤。直觉上应该过滤掉 "the"、"is"、"and" 之类的高频词。但实验发现，对于短文本（Agent 记忆条目通常 50-500 词），停用词的分布反而携带了文体信息——代码片段和自然语言描述的停用词分布完全不同。加了停用词过滤后，"Python 循环写法对比" 和一段 Python 代码的相似度反而下降了。保留它们，让余弦自己降权高频词。

词汇维度弥补了标签维度的不足，但它有自己的盲区：同义词。"mock" 和 "stub" 是同义的不同词。TF 矩阵把它们当作完全独立的维度，无法识别它们的语义关系。这需要第三维。

四、第三维：语义相似度 — embedding_distance

这是唯一需要外部模型的维度。用 sqlite-vec 存储 128 维 Embedding，余弦距离查询：

-- sqlite-vec 语义搜索
SELECT key, distance
FROM entries_vec
WHERE embedding MATCH :query_embedding
  AND k = 10
ORDER BY distance;

语义维度最强大——它知道 "mock" 和 "stub" 相关，知道 "pytest" 和 "unittest" 类似。但它也有弱点：

  • 短文本表现差：5 个词的记忆条目，Embedding 向量信息量太低，噪声大
  • 计算成本高：需要 Embedding 模型推理，延迟 50-200ms
  • 语义漂移：不同模型对同一文本给出不同 Embedding，迁移成本高

五、三维组合：涌现能力

当三个维度都到位后，一些单维度做不到的能力自然涌现。

5.1 复合相似度打分

最直接的用法是加权融合三个维度：

def composite_similarity(store, key1, key2, weights=(0.3, 0.3, 0.4)):
    """三维度加权复合相似度"""
    w_tag, w_lex, w_sem = weights
    s_tag = store.tag_jaccard(key1, key2) or 0.0
    s_lex = store.content_similarity(key1, key2) or 0.0
    s_sem = 1.0 - store.embedding_distance(key1, key2)  # distance → similarity
    return w_tag * s_tag + w_lex * s_lex + w_sem * s_sem

权重要根据场景调。标签密集的知识库给标签维度更高权重；短文本多的场景降低语义维度权重。这里没有银弹。

5.2 去重策略

三个维度同时高于阈值 = 几乎确定是重复：

def is_likely_duplicate(store, key1, key2):
    """三维度去重判定"""
    tag_sim = store.tag_jaccard(key1, key2) or 0
    lex_sim = store.content_similarity(key1, key2) or 0
    sem_sim = 1.0 - (store.embedding_distance(key1, key2) or 1.0)

    # 三个维度同时高 → 确定性重复
    if tag_sim > 0.7 and lex_sim > 0.6:
        return True, "high-confidence duplicate"

    # 语义高度相似 + 标签完全相同 → 跨语言/改写重复
    if tag_sim == 1.0 and sem_sim > 0.92:
        return True, "semantic rewrite detected"

    # 仅词汇相似度极高 → 可能是模板生成
    if lex_sim > 0.85:
        return True, "near-identical text"

    return False, ""

这个策略比任何单维度去重都准确。纯语义去重会误杀"同主题不同结论"的条目，纯词汇去重会漏掉改写后的重复。

5.3 信息论质量评估

三个维度还带来了一个意外的能力：评估记忆库的"健康状况"。用 Shannon 信息熵：

import math
from collections import Counter

def tag_entropy(store):
    """标签分布的 Shannon 熵（bits）"""
    tag_counts = Counter()
    for entry in store.entries.values():
        for tag in entry.tags:
            tag_counts[tag] += 1

    total = sum(tag_counts.values())
    if total == 0:
        return 0.0

    entropy = 0.0
    for count in tag_counts.values():
        p = count / total
        entropy -= p * math.log2(p)

    return entropy

def max_entropy(store):
    """理论最大熵：所有标签均匀分布"""
    n = len(set(t for e in store.entries.values() for t in e.tags))
    return math.log2(n) if n > 0 else 0.0

# 使用：熵/最大熵的比值越接近 1.0，标签分布越健康
# 低于 0.5 说明少数标签垄断了知识库

标签熵低 = 少数标签被过度使用 = 知识库变成了垃圾堆。标签熵高 = 分类多样化 = 知识组织良好。这是一个简单但有效的知识库健康指标，而且只需要标签数据就能算。

更进一步，把信息论扩展到语义维度——Embedding 分布熵：

def embedding_diversity(store, bins=10):
    """Embedding 相似度分布的 Shannon 熵

    将所有 embedding 对的余弦相似度分到 10 个桶 [0,0.1)...[0.9,1.0]
    计算桶分布的熵。高熵 = 内容多样化；低熵 = 内容同质化。
    """
    keys = [k for k, e in store.entries.items() if e.embedding]
    n = len(keys)
    if n < 2:
        return 0.0

    # 所有点对的余弦相似度
    sims = []
    for i in range(n):
        for j in range(i + 1, n):
            sim = 1.0 - store.embedding_distance(keys[i], keys[j])
            sims.append(sim)

    # 分桶统计
    bucket_counts = [0] * bins
    for sim in sims:
        idx = min(int(sim * bins), bins - 1)
        bucket_counts[idx] += 1

    total = sum(bucket_counts)
    entropy = 0.0
    for count in bucket_counts:
        if count > 0:
            p = count / total
            entropy -= p * math.log2(p)

    return entropy

语义熵 + 标签熵，两个维度交叉看就能诊断知识库问题的根因：

标签熵语义熵诊断
高高✅ 健康状态：话题广泛，分类合理
低高⚠️ 标签不足：内容多样但分类粗糙
高低⚠️ 同质化：标签多但内容雷同
低低🔴 垃圾堆：需要全面清理

六、工程教训

从这次"相似度三角"的构建过程中，我得到几条不太显然的教训：

1. 先做最简单的维度。Tag Jaccard 的实现不到 20 行代码，但它立刻暴露了"仅靠标签不够"的需求。如果一开始就跳到 Embedding 维度，可能永远不会意识到标签维度作为 baseline 的价值——它会"just work"，但你不知道它漏了什么。

2. 浮点比较必须用容差。实现 content_similarity 时，同一段文本自比的余弦相似度不是 1.0，而是 0.9999999999998。这不是 bug——是 IEEE 754 浮点运算的固有特性。测试要用 abs(score - 1.0) < 1e-9，不能用 ==。这个坑我踩过不止一次。

3. sqlite-vec 在插入时强制维度。写测试时用了 3 维向量，结果插入直接报错。sqlite-vec 在 INSERT 时检查维度，不是在查询时。这意味着你的测试向量必须匹配生产维度（128 维），否则连写入都做不到。

4. 信息论指标比统计指标更有洞察力。tag_frequency 告诉你 "python 出现了 47 次"，但 tag_entropy 告诉你 "你的标签分布偏离均匀分布 0.3 bits"。前者是数据，后者是诊断。当你设计 analytics API 时，优先实现能给出诊断信号的指标。

七、下一步：TF-IDF 与关键词提取

相似度三角完成后，一个自然的延伸是 TF-IDF 关键词提取。把整个记忆库当作文档语料，对每条记忆算每个词的 TF-IDF 分数：

def content_keywords(store, key: str, top_n: int = 10):
    """TF-IDF 关键词提取：哪些词让这条记忆在语料中与众不同"""
    target = store._get(key)
    if not target or not target.content:
        return []

    import re, math
    from collections import Counter

    # 目标条目的词频
    words = re.findall(r'\w+', target.content.lower())
    tf = Counter(words)

    # 语料级的文档频率
    N = len(store.entries)
    all_words = set(words)
    df = {}
    for w in all_words:
        df[w] = sum(
            1 for e in store.entries.values()
            if e.content and w in e.content.lower()
        )

    # TF-IDF 打分
    scored = []
    for word, freq in tf.items():
        idf = math.log((N + 1) / (df.get(word, 0) + 1)) + 1  # 平滑 IDF
        score = freq * idf
        scored.append((word, score))

    scored.sort(key=lambda x: x[1], reverse=True)
    return scored[:top_n]

关键词提取的价值在于：它告诉你每条记忆的"信息指纹"。在一个有 1000 条记忆的库里，"python" 这个词可能出现在 300 条中（IDF 很低），而 "coroutine" 只出现在 5 条中（IDF 很高）。TF-IDF 会自动把 "coroutine" 排到前面——它是这条记忆的辨识度信号。

这个能力可以用于自动标签推荐：提取 top-N 关键词作为候选标签，让 Agent 确认。比让 Agent 从头想标签更高效，也比完全自动打标签更可控。

总结

相似度不是一个概念，而是三个正交维度的组合。分类相似度捕捉显式标注的意图，词汇相似度捕捉表层文本的重叠，语义相似度捕捉深层概念的关联。单独使用任何一个都有盲区，组合使用后涌现出的能力——精确去重、复合打分、知识库健康诊断——是单维度无法实现的。

如果你在构建 Agent 记忆系统，别只做 Embedding。先把三个维度都搭起来，然后根据你的数据特征调权重。信息论指标（熵）是评估系统健康的好工具，优先于原始统计量。
