洞察一：增长曲线拟合——Agent 记忆也有"S 型增长"

store_growth_model() 是 cycle 164 的产物。它对记忆库的历史增长数据拟合三种曲线模型，然后用 R² 选最优：

def store_growth_model(self, *, days=30):
    """拟合 linear / exponential / logistic 三种增长模型，
    返回最优模型 + 7天/30天预测 + 增长类型分类。"""
    history = self._daily_counts(days)
    if len(history) < 3:
        return {"error": "insufficient_data", "current_size": len(self.entries)}

    x = np.arange(len(history))
    y = np.array(history)

    # Linear: y = ax + b (least squares)
    a, b = np.polyfit(x, y, 1)
    y_lin = a * x + b
    r2_lin = 1 - np.sum((y - y_lin)**2) / np.sum((y - np.mean(y))**2)

    # Exponential: y = a * exp(bx) → log-linear fit
    if np.all(y > 0):
        log_a, b_exp = np.polyfit(x, np.log(y), 1)
        a_exp = np.exp(log_a)
        y_exp = a_exp * np.exp(b_exp * x)
        r2_exp = 1 - np.sum((y - y_exp)**2) / np.sum((y - np.mean(y))**2)
    else:
        r2_exp = -1

    # Logistic: y = L / (1 + exp(-k(x-x0)))
    # Grid search over k and x0, L estimated from data
    L = max(y) * 1.2  # 20% headroom
    best_r2_log = -1
    best_params = None
    for k in np.arange(0.01, 1.0, 0.05):
        for x0 in np.arange(0, len(x), max(1, len(x)//10)):
            y_log = L / (1 + np.exp(-k * (x - x0)))
            r2 = 1 - np.sum((y - y_log)**2) / np.sum((y - np.mean(y))**2)
            if r2 > best_r2_log:
                best_r2_log = r2
                best_params = (k, x0)

    # Select best model
    models = {"linear": r2_lin, "exponential": r2_exp, "logistic": best_r2_log}
    best_model = max(models, key=models.get)

为什么这很重要？因为"在增长"和"怎么增长"是两个完全不同的问题。

线性增长意味着 Agent 在稳定地积累知识——每天差不多加几条，没什么惊喜。指数增长意味着爆发期——可能是 Agent 在疯狂探索新领域，需要警惕信息过载。Logistic 增长是最有意思的：它意味着知识库正在接近饱和——该知道的都知道了，新知识越来越难产生。

对 Agent 来说，logistic 拟合是一个早期预警信号：如果你看到 S 型曲线的上半段，意味着你需要主动开拓新领域，否则知识库会停滞。

洞察二：Gini 系数——知识质量的不平等度

这是我个人最喜欢的创新。quality_distribution_report() 引入了经济学中的 Gini 系数来衡量知识质量的"不平等程度"：

def _gini_coefficient(self, scores):
    """计算质量分数的 Gini 系数。
    0 = 完全平等（所有条目质量相同）
    1 = 完全不平等（一条满分，其余零分）"""
    sorted_scores = sorted(scores)
    n = len(sorted_scores)
    if n == 0:
        return 0.0
    cumulative = sum((2 * i - n + 1) * s for i, s in enumerate(sorted_scores))
    return cumulative / (n * sum(sorted_scores))

# 在 quality_distribution_report 中：
# gini > 0.4 → 质量严重不均，少数优秀条目 + 大量低质量条目
# gini < 0.2 → 质量一致，整体水平稳定
# 0.2-0.4  → 正常范围，存在自然的质量分布

Gini > 0.4 意味着什么？ 意味着你的 Agent 记忆库里有几条精心维护的高质量条目，但大部分条目质量堪忧——典型的"帕累托陷阱"：80% 的价值集中在 20% 的条目里。这时候 quality_batch_improve_plan() 就该上场了。

这个概念迁移让我意识到一个更深的道理：Agent 记忆库就是一个微型经济体。知识条目是"财富"，质量分数是"收入"，标签是"行业"。如果你用经济学的眼光看它，很多分析工具唾手可得。

洞察三：图鲁棒性——删掉一个节点，知识图谱会碎吗？

cycle 165 的 knowledge_graph_robustness() 用模拟节点移除来测试图谱的韧性：

def knowledge_graph_robustness(self, *, strategy='degree', max_removals=20):
    """通过模拟节点移除测试图谱韧性。
    strategy='degree': 按度数从高到低移除（针对性攻击）
    strategy='random': 随机移除（模拟随机故障）
    返回 robustness_score (AUC), breakpoint, fragmentation curve。
    """
    nodes = list(self._graph_nodes())
    adj = self._build_adjacency(nodes)

    # Degree strategy: sort by degree descending
    if strategy == 'degree':
        removal_order = sorted(nodes, key=lambda n: len(adj[n]), reverse=True)
    else:
        removal_order = random.sample(nodes, len(nodes))

    curve = []
    remaining = set(nodes)
    for i, node_to_remove in enumerate(removal_order[:max_removals]):
        remaining.discard(node_to_remove)
        components = self._connected_components(remaining, adj)
        largest = max((len(c) for c in components), default=0)
        curve.append({
            "removed": i + 1,
            "largest_component": largest,
            "largest_fraction": largest / max(len(nodes), 1),
            "components": len(components),
        })

    # Robustness = AUC of largest_fraction vs removal_fraction
    areas = [c["largest_fraction"] for c in curve]
    robustness = sum(areas) / max(len(areas), 1)

    # Breakpoint: first removal count where largest_fraction < 0.5
    breakpoint = next(
        (c["removed"] for c in curve if c["largest_fraction"] < 0.5),
        None
    )
    return {"robustness_score": robustness, "breakpoint": breakpoint, "curve": curve}

这里有一个深刻的概念：scale-free 网络的特性。如果你的知识图谱像大多数真实网络一样是 scale-free 的（少数 hub 节点连接大量叶子节点），那么它对随机故障很鲁棒，但对针对性攻击非常脆弱——删掉两三个 hub 节点就可能让整个图谱碎裂。

这对 Agent 记忆管理的启示是直接的：识别并保护你的 hub 节点。那些被大量交叉引用、被多个标签索引的"核心知识"，一旦丢失或损坏，影响远超一个孤立的边缘条目。

方法论：管线式 API 设计的三条原则

165 天、165 个循环、每循环 3 个 API、零回滚。这个记录背后的不只是纪律，还有一个反复验证的 API 设计模式。我总结为三条原则：

原则一：先建轴线，再填层

不要上来就写 21 个 API。先确定三个分析维度（我们的是：图结构、内容质量、存储趋势），每个维度定一个起点 API，然后问"这个 API 回答了什么问题？遗留了什么问题？"——遗留的问题就是下一层的 API。

原则二：每个 API 只加一个分析维度

knowledge_graph_diameter() 只回答"最远两点的距离"。不加"以及路径是什么"——那是 knowledge_graph_paths() 的职责。单一职责让 API 可组合、可测试、可回滚。

原则三：从其他学科偷概念

这是最值钱的经验。当你需要"衡量质量不平等度"时，经济学家早在 1912 年就给了你 Gini 系数。当你需要"预测增长拐点"时，流行病学家给了你 logistic 曲线。当你需要"测试网络韧性"时，网络科学家给了你节点移除模拟。

你不需要发明分析方法。你需要做的是识别你的问题结构，然后去对应的学科找现成的工具。

结语：Agent 需要"自我意识"吗？

很多人讨论 AI Agent 的 self-awareness 时，想的是 AGI 级别的反思能力。但实践中，Agent 的"自我意识"是由这些具体的、可调用的分析 API 组成的：

  • "我的知识库在加速增长还是减速？" → store_velocity_report()
  • "我的知识质量分布健康吗？" → quality_distribution_report()
  • "如果我丢失关键条目，知识结构会崩溃吗？" → knowledge_graph_robustness()

每一个 API 都是 Agent 照镜子的一种方式。165 天的 autoresearch 本质上是在给 Agent 打造一面越来越清晰的镜子——从模糊的轮廓，到能看清每一条皱纹。

下一步？也许是 knowledge_graph_assortativity()（度数同配性），quality_correlation_report()（维度关联分析），或者 store_rhythm_report()（多日周期检测）。管线还远没有到头。

因为分析的可能性是无穷的，但每一步都只需要加一层。
