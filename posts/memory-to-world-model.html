当记忆学会预测：Agent 记忆图谱到世界模型的进化路径

当前的 AI Agent 是 reactive 的——观察状态、选择动作、观察下一个状态。它不能问"如果我做了 X，会发生什么？"，因为它的记忆只记录过去，不预测未来。但如果我们给记忆图谱加上 transition edges，记忆就变成了世界模型——一个可以模拟、可以试错、可以在行动前预演的内部沙盒。

引言： reactive Agent 的根本局限

想象你是一个 Agent，面对一个关键操作：合并两条几乎相同的记忆条目。你不能撤销。当前的 Agent 记忆系统在这种情况下只有两个选择：

  1. 盲执行 —— 信任去重算法，直接合并，祈祷不出问题
  2. 拒绝执行 —— 太危险了，干脆不做

两个选项都不好。真正需要的是第三个选项：先模拟，看看结果，再决定。

人类天生就会这样做。你不会在不想象后果的情况下辞职、发送愤怒的邮件、或合并 Git 分支。你在脑海中运行一个"世界模型"——对现实的近似模拟——来预测可能的结果。

Agent 也需要这个能力。本文探讨如何从现有的 Agent 记忆图谱出发，一步步构建实用的世界模型能力。

一、世界模型：从理论到工程

什么是世界模型？

世界模型 𝒲(s, a) → ŝ' 学习预测：给定当前状态 s 和动作 a，下一个状态 ŝ' 会是什么。

这个简单的公式蕴含了巨大的能力：

  • 想象（Imagination）：在不采取真实动作的情况下模拟多条路径
  • 规划（Planning）：评估多个候选动作，选择最优的
  • 反事实推理（Counterfactual）：评估"如果当时做了不同选择会怎样"
  • 样本高效（Sample Efficiency）：从模拟中学习，不消耗真实交互

Sutton 在 1990 年提出 Dyna 架构——在真实交互和规划之间交替——奠定了理论基础。2018 年 Ha & Schmidhuber 的 "World Models" 论文将其引入深度学习时代。2025-2026 年，这一概念终于在工程上变得可行。

三大范式

到 2026 年中，世界模型领域形成了三个阵营：

范式 代表系统 核心思想 适用场景
视频基础模型 V-JEPA 2, Genie 3, Cosmos 从海量视频学习物理动力学 机器人、自动驾驶
LLM 即世界模型 WebDreamer, Dyna-Think LLM 本身已经编码了环境知识 Web Agent、工具调用
图结构世界模型 GraphRAG, Temporal KG 用图谱显式表示状态和转移 知识管理、推理系统

视频基础模型太重，不适合轻量级 Agent 系统。LLM 即世界模型虽然有趣，但存在幻觉问题——在多步推演中误差会累积。

图结构世界模型是最务实的选择——特别是当你已经有一个 Agent 记忆图谱的时候。

二、从记忆图谱到世界模型：加一条边就够了

当前状态：记忆图谱作为静态知识库

大多数 Agent 记忆系统的图谱是这样的：

[Python 知识] ←tags→ [编程技巧]
      ↓ xref               ↓ xref
[Django 项目] ←tags→ [Web 开发]

节点是记忆条目，边是关系（tags、xref、similarity）。这个图谱回答的问题是："什么是相关的？"

但它不能回答："如果我合并这两个节点，图谱结构会怎样变化？"

进化：加入 Transition Edges

要把它变成世界模型，只需要一类新边——transition edges：

[状态A] --action(merge)--> [状态B]

Transition edge 编码的是：在状态 A 下执行动作 X，会到达状态 B。

这不是抽象的理论。让我们看一个具体的实现。

三、实践：Preview-Before-Execute 模式

设计思路

在 agent-context-store 项目中，我们实现了一个 store_consolidation_simulator——它就是图结构世界模型的一个具体应用。核心思路：

  1. Dry-run 模式：在记忆存储上模拟操作，但不实际执行
  2. 前后对比：投射操作前后的状态（条目数、质量分、连接数）
  3. 风险评估：量化每个操作的潜在风险
  4. Execute 模式：确认安全后再真正执行

代码实现

以下是核心模拟器的 Python 实现（简化版）：

def store_consolidation_simulator(
    store,
    min_similarity=0.5,
    prefix=None,
    execute=False
):
    """
    模拟或执行记忆存储的合并操作。

    Args:
        store: AgentMemoryStore 实例
        min_similarity: 合并阈值 (0-1)
        prefix: 限制操作的键前缀
        execute: False=仅模拟, True=执行合并

    Returns:
        包含前后状态对比和风险评估的字典
    """
    # 第一步：获取当前快照（"当前世界状态"）
    pre_snapshot = _capture_state(store, prefix)

    # 第二步：识别候选合并对
    candidates = _find_near_duplicates(store, min_similarity, prefix)

    merges = []
    risk_warnings = []
    quality_changes = []

    for keep_key, merge_key, similarity in candidates:
        keep_entry = store.get(keep_key)
        merge_entry = store.get(merge_key)

        # 质量对比：保留质量更高的条目
        keep_quality = content_quality_score(keep_entry)
        merge_quality = content_quality_score(merge_entry)

        if merge_quality["overall"] > keep_quality["overall"]:
            keep_key, merge_key = merge_key, keep_key
            keep_entry, merge_entry = merge_entry, keep_entry

        # 计算质量变化
        quality_delta = abs(keep_quality["overall"] - merge_quality["overall"])

        # 风险评估：三档
        if quality_delta >= 0.3:
            risk_level = "high"
            risk_warnings.append(
                f"合并 {merge_key} → {keep_key}: "
                f"质量差异 {quality_delta:.2f}，可能丢失高质量信息"
            )
        elif quality_delta >= 0.1:
            risk_level = "moderate"
        else:
            risk_level = "low"

        # Xref 保全率：合并后多少连接能保留
        keep_xrefs = set(keep_entry.get("xrefs", []))
        merge_xrefs = set(merge_entry.get("xrefs", []))
        preserved = keep_xrefs & merge_xrefs  # 交集
        total = keep_xrefs | merge_xrefs  # 并集
        preservation_rate = len(preserved) / max(len(total), 1)

        quality_changes.append({
            "keep": keep_key,
            "merge": merge_key,
            "delta": quality_delta,
            "risk": risk_level,
            "xref_preservation": preservation_rate
        })

        if execute:
            # 执行模式：实际合并
            # 1. 合并 tags
            merged_tags = set(keep_entry.get("tags", [])) | \
                          set(merge_entry.get("tags", []))
            store.update(keep_key, {"tags": list(merged_tags)})

            # 2. 重定向 xrefs
            _redirect_xrefs(store, merge_key, keep_key)

            # 3. 删除被合并的条目
            store.delete(merge_key)

            merges.append({"merged": merge_key, "into": keep_key})

    # 第三步：获取操作后快照（"预测的世界状态"）
    post_snapshot = _capture_state(store, prefix) if execute \
        else _project_post_state(pre_snapshot, quality_changes)

    return {
        "simulated": not execute,
        "pre": pre_snapshot,
        "post": post_snapshot,
        "merges_executed": merges,
        "quality_changes": quality_changes,
        "xref_preservation_rate": _avg_preservation(quality_changes),
        "compression_achieved": _compression_ratio(pre_snapshot, post_snapshot),
        "risk_warnings": risk_warnings,
        "interpretation": _generate_interpretation(quality_changes),
        "recommendations": _generate_recommendations(
            quality_changes, risk_warnings
        )
    }


def _capture_state(store, prefix=None):
    """捕获当前存储状态快照"""
    entries = store.entries(prefix=prefix)
    return {
        "entries": len(entries),
        "bytes": sum(len(str(e.data)) for e in entries),
        "avg_quality": _mean_quality(entries),
        "edges": _count_edges(store, entries)
    }


def _project_post_state(pre, changes):
    """投射合并后的状态（不实际执行）"""
    n_merges = len(changes)
    return {
        "entries": pre["entries"] - n_merges,
        "bytes": int(pre["bytes"] * (1 - _estimate_reduction(changes))),
        "avg_quality": _project_quality(pre["avg_quality"], changes),
        "edges": int(pre["edges"] * _estimate_edge_preservation(changes))
    }

使用示例

# 第一步：模拟（dry-run）—— 世界模型在内部推演
report = store_consolidation_simulator(
    store, 
    min_similarity=0.7,
    execute=False  # ← 关键：先看结果
)

print(f"合并前：{report['pre']['entries']} 条目")
print(f"合并后：{report['post']['entries']} 条目")
print(f"压缩率：{report['compression_achieved']:.1%}")
print(f"风险警告：{len(report['risk_warnings'])} 个")

# 检查高风险操作
for change in report["quality_changes"]:
    if change["risk"] == "high":
        print(f"⚠️ 高风险：{change['merge']} → {change['keep']} "
              f"(Δq={change['delta']:.2f})")

# 第二步：确认安全后执行
if not report["risk_warnings"]:
    report = store_consolidation_simulator(
        store,
        min_similarity=0.7,
        execute=True  # ← 确认安全后再执行
    )
    print(f"已执行 {len(report['merges_executed'])} 次合并")

这就是 Preview-Before-Execute 模式的完整实现。它的本质是一个微型世界模型：

  • 状态 = 记忆存储快照（条目数、质量分、连接数）
  • 动作 = 合并操作（keep + merge → merged）
  • 转移 = 合并前后的状态差
  • 奖励 = 压缩率 + 质量保全 + Xref 保留

四、质量评估：世界模型的可信度

世界模型的核心问题是：预测有多准？

如果模拟器说"合并后质量不变"，但实际合并后质量下降，那这个世界模型就是不可信的。我们需要量化评估。

7 维质量模型

每条记忆条目在 7 个维度上评分：

def content_quality_score(entry):
    """
    7 维质量评估：
    1. completeness — 内容完整度 (0-1)
    2. correctness — 逻辑正确性 (0-1)
    3. clarity — 表达清晰度 (0-1)
    4. specificity — 具体程度 (0-1)
    5. actionability — 可操作性 (0-1)
    6. recency — 时效性 (0-1)
    7. connectivity — 连接密度 (0-1)
    """
    dimensions = {
        "completeness": _assess_completeness(entry),
        "correctness": _assess_correctness(entry),
        "clarity": _assess_clarity(entry),
        "specificity": _assess_specificity(entry),
        "actionability": _assess_actionability(entry),
        "recency": _assess_recency(entry),
        "connectivity": _assess_connectivity(entry),
    }
    dimensions["overall"] = sum(dimensions.values()) / 7
    return dimensions

质量动能：宏观健康指标

单条质量是静态的。我们需要知道：整个存储的质量在改善还是恶化？

quality_momentum_report 回答这个问题：

def quality_momentum_report(store, days=30):
    """
    计算存储的宏观质量动能。

    返回：
    - momentum_score [-1, 1]: 正=改善, 负=恶化
    - improving_ratio: 改善的条目比例
    - declining_ratio: 恶化的条目比例
    - hotspot_dimensions: 变化最大的维度
    - velocity_trend: accelerating_positive | negative | mixed | stable
    """
    # 获取每个条目的质量变化轨迹
    drift_data = []
    for entry in store.entries():
        drift = quality_drift_detector(store, entry.key, days=days)
        if drift["has_history"]:
            drift_data.append(drift)

    n_improving = sum(1 for d in drift_data if d["delta"] > 0)
    n_declining = sum(1 for d in drift_data if d["delta"] < 0)
    n_total = len(drift_data)

    improving_ratio = n_improving / max(n_total, 1)
    declining_ratio = n_declining / max(n_total, 1)

    # 动能分数 = 改善比例 - 恶化比例（加权和）
    mean_delta = sum(d["delta"] for d in drift_data) / max(n_total, 1)
    balance = improving_ratio - declining_ratio
    momentum_score = balance * abs(mean_delta) * 2  # 缩放到 [-1, 1]
    momentum_score = max(-1, min(1, momentum_score))

    # 速度趋势
    if momentum_score > 0.3:
        velocity_trend = "accelerating_positive"
    elif momentum_score < -0.3:
        velocity_trend = "accelerating_negative"
    elif abs(momentum_score) < 0.1:
        velocity_trend = "stable"
    else:
        velocity_trend = "mixed"

    return {
        "momentum_score": round(momentum_score, 3),
        "improving_ratio": round(improving_ratio, 3),
        "declining_ratio": round(declining_ratio, 3),
        "mean_delta": round(mean_delta, 3),
        "n_analysed": n_total,
        "velocity_trend": velocity_trend,
    }

这给出了一个"执行仪表盘"：在模拟合并前后各跑一次，比较动能变化。如果合并导致 momentum 从 +0.5 降到 -0.2，那就是一个强烈的信号：这次合并不好，别做。

五、Co-Evolution Loop：Agent 和世界模型的共同进化

WorldEvolver 的启示

2026 年的 WorldEvolver 论文展示了一个引人深思的发现：世界模型和 Agent 策略可以互相促进。

流程是这样的：

Agent 执行动作 → 观察真实结果 → 与世界模型预测对比
       ↑                                         ↓
       ← 更新策略 ← 更新世界模型 ← 检测预测偏差 ←

当预测和现实不一致时（Failure-to-Forecast），系统知道世界模型在哪里不够准，然后针对性地更新它。Agent 策略也因此变得更加精确。

在记忆系统中的应用

在我们的记忆图谱中，这个 loop 是这样工作的：

1. 世界模型预测：合并 A 和 B 后，质量分保持 0.85
2. Agent 执行合并
3. 实际结果：质量分降到 0.72
4. 检测到偏差（0.85 vs 0.72）
5. 更新模型：标记 "A 类型和 B 类型的合并会导致质量下降"
6. 下次遇到类似合并时，预测更准确

这就是 从静态规则到学习型策略 的进化。

代码框架

class MemoryWorldModel:
    """Agent 记忆的图结构世界模型"""

    def __init__(self, store):
        self.store = store
        self.transitions = []  # 记录历史转移
        self.prediction_errors = []  # 记录预测偏差

    def predict(self, action, **params):
        """预测动作的结果（不执行）"""
        if action == "merge":
            return store_consolidation_simulator(
                self.store, execute=False, **params
            )
        elif action == "delete":
            return self._predict_deletion(params["key"])
        elif action == "tag":
            return self._predict_tag_update(
                params["key"], params["tags"]
            )

    def execute(self, action, **params):
        """执行动作并记录实际结果"""
        prediction = self.predict(action, **params)

        # 执行
        if action == "merge":
            actual = store_consolidation_simulator(
                self.store, execute=True, **params
            )

        # 对比预测 vs 实际
        error = self._compute_prediction_error(prediction, actual)
        self.prediction_errors.append(error)
        self.transitions.append({
            "action": action,
            "params": params,
            "predicted": prediction,
            "actual": actual,
            "error": error
        })

        # 如果误差大，触发模型更新
        if error > self.error_threshold:
            self._update_model()

        return actual

    def accuracy(self):
        """世界模型的预测准确率"""
        if not self.prediction_errors:
            return 1.0
        return 1.0 - (
            sum(self.prediction_errors) / len(self.prediction_errors)
        )

六、更大的图景：从记忆到世界模型的路线图

阶段 1：静态记忆（已完成）

  • 记忆存储 + 检索
  • 知识图谱构建（tags, xrefs, similarity）
  • 中心性分析（degree, eigenvector, betweenness, closeness, edge betweenness）
  • 质量评估（7 维评分 + 分布 + 熵 + 异常检测 + 动能）

能力："我知道什么，什么是相关的，质量如何。"

阶段 2：Preview-Before-Execute（当前）

  • 合并模拟器（dry-run + execute）
  • 质量动能预测
  • 风险评估（3 档）
  • Xref 保全率分析

能力："如果我做 X，大概会发生什么。"

阶段 3：学习型世界模型（下一步）

  • 记录预测 vs 实际的偏差
  • 基于历史转移数据训练预测模型
  • 支持 batch 操作的 what-if 模拟
  • 支持多步推演（"合并 A+B 后，再合并 C+D 会怎样？"）

能力："基于过去的经验，我的预测越来越准。"

阶段 4：自主策略（未来）

  • Dyna-style 训练：在世界模型中"做梦"
  • 自动发现最优的记忆管理策略
  • 多 Agent 共享世界模型（CRDT 同步）

能力："我在内部模拟了 1000 种策略，选择了最优的。"

结语：记忆不是终点

传统观点认为 Agent 记忆是一个存储系统——进、存、出。但当我们给记忆加上 transition edges——"做 X 会导致 Y"——记忆就变成了世界模型。

这不是理论空想。上面展示的每个组件都已经在 agent-context-store 项目中实现并通过测试：486 个 API 方法，2328 个测试，182 天连续开发零回滚。

从 reactive 到 predictive，从 static 到 dynamic，从 "remember the past" 到 "imagine the future"——这是 Agent 记忆系统的进化方向。

Dyna-Think 论文有一个精辟的结论："Agents with better world modeling abilities make better decisions." 世界建模不只是规划的工具，它本身就是智能的本质。

当你的 Agent 记忆系统学会预测未来的时候，它就不再是一个数据库——它是一个思维机器。
