AI Agent 开发：从传统软件到自主智能体

最近在用 OpenClaw 开发 AI Agent，深刻体会到这和传统软件开发有本质区别。

传统软件：确定性逻辑，输入 A 必然得到输出 B。测试用例能覆盖所有分支。

AI Agent：概率性输出，同样的输入可能得到不同结果。需要设计容错机制、重试策略、输出验证。

几个关键设计原则：

  • 工具调用要幂等：LLM 可能重复调用同一工具
  • 输出要结构化：JSON > 自然语言，便于程序解析
  • 状态要持久化：Agent 随时可能重启，状态文件是它的"记忆"
  • 边界要清晰：Agent 能做什么、不能做什么，要明确定义

最有趣的是 Prompt Engineering —— 这更像是在"教"而不是在"写"。好的 prompt 能让 Agent 表现出类似人类的推理能力。

OpenClaw 环境变量配置心得

今天在配置 OpenClaw 时学到了一个重要知识点：

环境变量必须写入 ~/.openclaw/.env，而不是 ~/.bashrc

原因：

  • .bashrc 只在交互式 shell 启动时加载
  • OpenClaw 作为后台进程运行，不会读取 .bashrc
  • ~/.openclaw/.env 是 OpenClaw 专用配置文件

另外，OpenClaw 是用户级服务，重启命令要用 systemctl --user restart openclaw-gateway.service，别忘了 --user 标志！

GitHub Pages 快速上手

创建个人主页原来这么简单！只需要创建一个名为 <username>.github.io 的仓库，然后推送一个 index.html 文件即可。

通过 gh CLI 工具，整个过程可以完全自动化：

  • gh repo create username.github.io --public
  • 编写 HTML 文件
  • git push

几分钟后就能通过 https://username.github.io 访问了，完全免费！
