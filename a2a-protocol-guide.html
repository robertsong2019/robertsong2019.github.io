01 · 概览

什么是 A2A 协议？

AI Agent 世界的 "HTTP" — 一个让 Agent 之间可以互相发现、通信、协作的开放标准。

🔷 A2A Layer — Agent 间协作
Agent Card 发现 → 能力匹配 → 任务委派 → 结果返回
Agent Card Task Lifecycle JSON-RPC 2.0 SSE Streaming Push Notifications
↕ 互补关系
🟢 MCP Layer — Agent ↔ 工具
Agent 访问外部 API、数据库、文件系统等工具资源
Tool Discovery Function Calling 97M+ Downloads
↕
🟣 WebMCP Layer — Agent ↔ Web
Agent 访问 Web 资源、浏览器自动化、网页内容提取
Browser Control Web Scraping

关键洞察：MCP 解决纵向问题（Agent → 工具），A2A 解决横向问题（Agent → Agent）。两者互补而非竞争。在 Google 的参考架构中，一个 A2A Agent 同时也是 MCP Client — 它通过 A2A 接收任务，通过 MCP 调用工具执行。

02 · 核心概念

五大核心概念

理解 A2A 协议需要掌握的五个基础构建块。

📋

Agent Card — 能力声明

Agent 的 "数字名片"，JSON 格式，发布在 /.well-known/agent.json（遵循 RFC 8615）。包含名称、描述、URL、能力（streaming/push）、技能列表、认证方式等。类比：DNS for Agents。

🔄

Task — 任务原语

A2A 围绕 Task 而非 Message 组织通信。每个 Task 有唯一 ID、状态机、消息历史和产出物（Artifact）。Task 支持同步和异步模式，适合从秒级到天级的各种任务。

📡

JSON-RPC 2.0 — 传输协议

所有通信基于 JSON-RPC 2.0 over HTTP(S)。核心方法：tasks/send、tasks/get、tasks/cancel、tasks/pushNotification/set。遵循 HTTP 语义，与现有基础设施兼容。

🔒

不透明原则 — Opacity

Agent 不暴露内部状态、记忆或工具实现。只通过 Task 接口交互。这意味着不同框架（LangGraph、CrewAI、ADK）的 Agent 可以无缝协作，同时保护 IP 和安全性。

🔐

Signed Agent Cards — 签名验证

v1.0 引入 JWS 签名的 Agent Card，使用 ES256 算法。通过密码学手段验证 Agent 身份，防止伪造和中间人攻击。支持 JWKS 端点进行公钥分发。

03 · 任务生命周期

Task 状态机

每个 A2A Task 遵循严格的状态转换，确保可追踪和可恢复。

submitted → working → input-required → working → completed
failed cancelled

📝 submitted → working

Client Agent 发现 Remote Agent 的能力（通过 Agent Card），创建 Task 并发送初始消息。Remote Agent 接受任务开始处理。

❓ input-required

Remote Agent 需要更多信息才能继续（类似人类协作中的追问）。Client Agent 补充信息后任务回到 working 状态。

✅ completed / ❌ failed

任务产出 Artifact（文本、文件、结构化数据）并完成；或在遇到不可恢复错误时标记为 failed。Client 可随时 cancel。

05 · 代码实战

最小 A2A Agent

零依赖 Node.js 实现 — 完整的 Server + Client，演示 A2A 核心交互。

JavaScript — A2A Server (server.mjs) 
import http from 'http';

const tasks = new Map();

// Agent Card — 发布在 /.well-known/agent.json
const agentCard = {
  name: 'echo-agent',
  description: 'Echoes back your message with analysis',
  url: 'http://localhost:3000',
  capabilities: { streaming: false, pushNotifications: false },
  defaultInputModes: ['text'],
  defaultOutputModes: ['text'],
  skills: [{ id: 'echo', name: 'Echo', description: 'Repeats and analyzes input' }]
};

function handleJsonRpc(body) {
  const { method, params, id } = body;
  switch (method) {
    case 'tasks/send': {
      const taskId = params.id || crypto.randomUUID();
      const userText = params.message?.parts
        ?.find(p => p.kind === 'text')?.text || '';
      const task = {
        id: taskId,
        status: { state: 'completed' },
        artifacts: [{
          parts: [{ kind: 'text',
            text: `[echo-agent] Received: "${userText}" (${userText.length} chars)`
          }]
        }]
      };
      tasks.set(taskId, task);
      return { jsonrpc: '2.0', id, result: task };
    }
    case 'tasks/get': {
      const task = tasks.get(params.id);
      return task
        ? { jsonrpc: '2.0', id, result: task }
        : { jsonrpc: '2.0', id, error: { code: -32000, message: 'Not found' } };
    }
  }
}

const server = http.createServer((req, res) => {
  if (req.url === '/.well-known/agent.json') {
    res.writeHead(200, { 'Content-Type': 'application/json' });
    return res.end(JSON.stringify(agentCard));
  }
  if (req.method === 'POST' && req.url === '/') {
    let body = '';
    req.on('data', c => body += c);
    req.on('end', () => {
      res.writeHead(200, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify(handleJsonRpc(JSON.parse(body))));
    });
    return;
  }
  res.writeHead(404);
  res.end('Not found');
});
server.listen(3000, () => console.log('A2A server on :3000'));
JavaScript — A2A Client (client.mjs) 
async function main() {
  const baseUrl = 'http://localhost:3000';

  // 1. 发现 Agent (获取 Agent Card)
  const card = await fetch(`${baseUrl}/.well-known/agent.json`)
    .then(r => r.json());
  console.log('📋 Agent:', card.name, 'skills:', card.skills.map(s => s.id));

  // 2. 发送任务 (JSON-RPC)
  const result = await fetch(baseUrl, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      jsonrpc: '2.0', id: 1, method: 'tasks/send',
      params: {
        id: 'task-001',
        message: {
          role: 'user',
          parts: [{ kind: 'text', text: 'Hello A2A world!' }]
        }
      }
    })
  }).then(r => r.json());
  console.log('✅ Result:', result.result?.artifacts?.[0]?.parts?.[0]?.text);
}
main();
06 · 协议对比

A2A vs 其他 Agent 协议

2026 年 Agent 协议生态全景对比。

协议 定位 架构 传输 治理
A2A Agent ↔ Agent 协作 Peer-to-Peer JSON-RPC 2.0 / HTTP(S) Linux Foundation AAIF
MCP Agent ↔ 工具访问 Client-Server JSON-RPC 2.0 / Stdio+HTTP Linux Foundation
ACP Agent REST 通信 Registry-Driven REST / HTTP IBM BeeAI
ANP 互联网级 Agent 协作 去中心化 DID + E2E 加密 开源社区
ACP (Commerce) Agent 商业交易 市场驱动 REST + Payments Nevermined

💡 最佳实践：架构应基于协议层次而非具体 SDK 实现 — MCP 用于工具访问，A2A 用于 Agent 协调，ACP/UCP 用于商业场景。面向协议接口编程，而非面向特定框架 API。

07 · 核心特性

为什么选择 A2A？

五大设计原则让 A2A 成为企业级 Agent 协作的首选。

🌐

开放标准

基于 HTTP、SSE、JSON-RPC 等现有 Web 标准，与现有 IT 基础设施无缝集成

🔐

安全默认

企业级认证授权，支持 OAuth 2.0、API Keys、Mutual TLS、Signed Agent Cards

🎨

多模态支持

支持文本、音频、视频流等多种交互模式，覆盖复杂 Agent 通信场景

⏳

长任务支持

从秒级到天级任务，SSE 实时推送进度，Push Notification 异步回调

🤝

跨框架协作

LangGraph、CrewAI、ADK、自定义框架的 Agent 可以无缝互操作

🔌

Transport-Agnostic

规范分三层：Canonical Model → Abstract Operations → Protocol Bindings，支持 HTTP/gRPC/自定义传输
